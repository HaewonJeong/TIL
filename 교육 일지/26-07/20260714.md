## 1. 연산자 우선순위 (`01_operator`)

파이썬 연산자의 우선순위를 공식 문서 기준으로 정리했다. (숫자가 작을수록 우선순위 높음)

| 순위 | 연산자 | 설명 |
|---|---|---|
| 1 | `()` | 괄호 |
| 2 | `**` | 거듭제곱 |
| 3 | `+x`, `-x`, `~x` | 단항 연산자 |
| 4 | `*`, `/`, `//`, `%` | 곱셈·나눗셈·몫·나머지 |
| 5 | `+`, `-` | 덧셈·뺄셈 |
| 6~9 | `<<`,`>>`,`&`,`^`,`\|` | 비트 연산 |
| 10 | `==`,`!=`,`>`,`<`,`is`,`in` 등 | 비교 연산자 |
| 11~13 | `not`, `and`, `or` | 논리 연산자 |
| 14 | `if ... else` | 조건 표현식 |
| 15 | `lambda` | 람다 표현식 |
| 16 | `:=` | 왈러스 연산자 |
| 17 | `=` | 대입 연산자 (최하위, 우→좌 결합) |

> 헷갈리는 수식은 항상 괄호로 우선순위를 명시하는 습관을 들이자.

## 2. 내장 함수 & 함수형 도구 (`01_builtins`)

- `dir(__builtins__)`: import 없이 바로 쓸 수 있는 내장 객체 목록 확인
- **`map(함수, iterable)`**: 각 요소에 함수를 적용한 결과를 반환 → 리스트 내포와 동일한 역할
  ```python
  lens = [*map(lambda x: len(x), texts)]
  list(map(lambda mid, final: (mid+final)/2, mid_terms, final_terms))  # 여러 iterable 동시 적용
  ```
- **`filter(함수, iterable)`**: 함수 결과가 `True`인 요소만 남김 → 리스트 내포의 `if` 조건과 동일
  ```python
  evens = list(filter(lambda x: x % 2 == 0, nums))
  ```
- **`reduce(함수, iterable, 초기값)`** (`functools` 모듈): 요소들을 누적 연산하여 단일 값 반환
  ```python
  from functools import reduce
  total = reduce(lambda acc, cur: acc + cur, nums, 0)   # 합계
  max_num = reduce(lambda a, b: a if a > b else b, numbers)  # 최댓값
  ```

## 3. 함수 (`01_fuction`)

- 함수는 반복 실행되는 절차적 코드 블록, **선언 후 호출**해서 사용
- **매개변수(parameter)** vs **매개인자(argument)**: 선언부의 변수 vs 호출부에서 전달하는 값
- **return**: 함수 실행을 마치고 값과 함께 호출부로 복귀

### 호출 방식
- 위치 인자: 선언 순서대로 대입 (`test(10, 20)`)
- 키워드 인자: 이름으로 대입, 순서 무관 (`test(a=1, b=2)`)
- 위치 인자는 반드시 키워드 인자보다 앞에 와야 함

### 기본값 매개변수
- 기본값이 없는 매개변수는 기본값이 있는 매개변수보다 **앞에** 선언해야 함

### packing / unpacking
| 구분 | 위치 | 문법 | 결과 타입 |
|---|---|---|---|
| packing | 함수 **선언부** | `*args` | tuple |
| packing | 함수 **선언부** | `**kwargs` | dict |
| unpacking | 함수 **호출부** | `*리스트` | 개별 인자로 풀림 |

- packing 매개변수는 일반 매개변수보다 **오른쪽**에 선언
- 전체 조합 예시: `def sample(a, b, c=99, *args, **kwargs)`
- `*lst, = zip(...)` 처럼 언패킹으로 변수에 바로 담을 수도 있음

### 1급 시민 객체로서의 함수
- 함수는 변수에 대입 가능, 다른 함수의 인자·반환값으로 사용 가능
  ```python
  doo = foo                 # 변수에 함수 대입
  runner(foo, 3)             # 함수를 인자로 전달
  def get_func():
      def bar(): ...
      return bar             # 함수를 반환값으로 사용(클로저)
  ```

## 4. 람다 함수 (`02_lambda`)

- `lambda 매개변수: 반환값` 형태의 **한 줄짜리 익명 함수**
  ```python
  f = lambda x: x ** 2
  add = lambda a, b=3: a + b        # 기본값도 사용 가능
  adder = lambda *args: sum(args)   # packing도 사용 가능
  ```
- 리스트 내포 안에서 즉시 실행하는 방식도 가능: `[(lambda x: x**2)(n) for n in range(1,11)]`
- 주로 `map`, `filter`, `sort(key=...)` 등에 짧은 함수가 필요할 때 활용

## 5. if 조건문 (`01_if_condition`)

- 파이썬은 **들여쓰기**로 코드 블록 구분
- **if**: 조건이 참일 때만 실행
- **if-else**: 참/거짓 둘 중 하나는 반드시 실행
- **if-elif-else**: 세 가지 이상의 경우의 수 처리 (예: 점수별 학점 A~F 산정)
- 범위 비교를 이어서 표현 가능: `0 <= score <= 100`
- `not(조건)`으로 조건 전체를 반전하여 유효성 검사에 활용

## 6. for 반복문 (`02_for_loop`)

- `for 변수 in iterable객체:` → 문자열, 리스트 등 반복 가능한 모든 객체 순회 가능
- **`range(start, end, step)`**: 반복 횟수/범위 지정
  ```python
  for n in range(2, 101, 2):   # 2~100 짝수
      print(n, end=' ')
  ```
- 실습: 1~100 홀수의 합 구하기, 2~9단 구구단 이중 for문으로 출력

## 7. while 반복문 (`03_while_loop`)

- 조건식이 **True인 동안 반복**, False가 되면 종료
- 반복 횟수를 사용자가 명확히 정할 수 없을 때 주로 사용
  ```python
  i = 0
  while i < 3:      # 조건식
      print('🍉')
      i += 1        # 증감식
  ```
- `while True:` + `break`로 조건에 따라 반복을 중간에 종료하는 패턴도 학습
  ```python
  while True:
      j += 1
      if j >= 3:
          break
  ```

## 8. 컴프리헨션 (`04_comprehension`)

- **리스트 내포**: `[표현식 for 변수 in 반복객체 if 조건]`
  ```python
  nums = [n + 100 for n in range(1, 101)]
  evens_sq = [n**2 for n in range(1, 101) if n % 2 == 0]
  ```
- **딕셔너리 내포**: `{키: 값 for ... in ...}`
  ```python
  target = {k: v for k, v in zip(students, scores)}
  ```
- for문 + append()로 작성한 코드를 한 줄로 압축한 것과 동일한 결과
- `filter()` / `map()`과 리스트 내포는 서로 대체 가능한 관계임을 확인
