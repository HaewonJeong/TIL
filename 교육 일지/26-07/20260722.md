##### Chatbot Memory
- 이전 대화를 기억하는 방법을 배운다. 
- 챗봇으로 이전 대화에서 필요한 정보를 저장, 요약, 삭제하고 다음 답변의 Context로 사용
- **RAG**가 외부 문서를 찾는 구조라면, **Memory**는 사용자와 나눈 대화의 맥락을 저장.
- GPT가 Context가 좀 더 길다. Claude는 짧다.
- Cluade에서 작업한것을 Codex로 통채로 들고 올 수 있는 플러그인이 있음. 혹은 인수인계 파일을 만들어주면 집어넣어서 그대로 사용 가능

#### 실습 코드 이해해보기 (01_chatbot_memory.py)
```
1. clean_korean_suffix (4~9줄)
"민수야"처럼 문장 끝에 붙는 "야"/"이야"를 떼어내는 함수예요. "민수야" → "민수"

2. ChatMemory.__init__ (14~18줄)
메모리가 담을 3개의 바구니를 만듭니다.

messages: 전체 대화 기록
profile: 이름/관심사/목표 같은 사용자 정보
summary: 요약 문장

3. add_message (20~25줄)
대화 한 마디(역할 + 내용 + 시간)를 messages 리스트에 추가.

4. recent_messages (27~28줄)
messages[-6:]처럼 뒤에서 N개만 잘라서 반환 — 프롬프트가 무한정 길어지지 않게 "최근 것만" 쓰는 용도.

5. remember_profile (30~43줄) ← 어제 말씀드린 버그가 있는 곳
문장에 특정 키워드가 있으면 그 뒤 내용을 잘라서 profile에 저장합니다. 문제는 "관심사"만 검사해서, 정보를 알려줄 때("관심사는 RAG야")도 걸리고 질문할 때("관심사가 뭐였지?")도 걸려버려요.

6. update_summary (45~65줄)
profile에 쌓인 값들을 문장으로 이어붙여 요약을 만듭니다. (진짜 서비스라면 이 요약을 LLM이 만들지만, 여기서는 그냥 문자열 조합으로 흉내만 낸 것)

7. context (67~74줄)
profile + summary + recent_messages 3개를 한 묶음으로 반환 — 실제로 LLM에 보낼 프롬프트를 만들 때 이 묶음을 그대로 끼워 넣게 됩니다 (7번 파일에서 이렇게 씀).

8. generate_reply (76~100줄)
사용자 메시지를 보고 "이름을 물어보는 건가?", "요약해달라는 건가?"를 if문으로 패턴 매칭해서 답을 고릅니다. 진짜 LLM 없이 규칙만으로 흉내 낸 챗봇이에요.

9. chat_once (102~106줄)
저장 → 답변 생성 → 답변 저장, 한 사이클.
```

#### 나머지 코드 AI 설명(참고)
```
01_chatbot_memory.py

오늘 살펴본 버전. remember_profile()에서 "관심사"를 조사 없이 느슨하게 검사해서, "관심사가 뭐였지?"라고 물으면 오답이 저장되는 버그가 있는 원본 파일.

02_memory_practice.py

같은 구조를 연습용으로 다시 옮겨 적은 파일. 처음부터 "관심사는"(조사 포함)으로 정확히 검사하도록 되어 있어 버그가 없고, 질문 판별 조건("내 관심사" + "뭐")도 더 엄격함. 01의 버그를 이미 고친 정답 버전에 가까움.

02_memory_scenarios.py

02_memory_practice.py를 실제 모듈로 불러와서(importlib), 아래 4가지 시나리오를 자동으로 돌려보고 결과를 출력하는 테스트 스크립트.

기본 기억
정보 변경 (이름을 바꿔 말하기)
없는 정보 질문
전체 profile 채우기
03_window_memory_lab.py

memory_core.py의 ChatMemory를 가져다가, window_size(기억할 최근 메시지 개수)를 2 / 4 / 6으로 바꿔가며 같은 10턴짜리 대화를 저장했을 때 "최근 대화"가 어떻게 잘려 보이는지 비교하는 실습.

04_profile_privacy_lab.py

사용자가 "이름을 잊어줘" / "모든 기억을 삭제해줘"라고 말하면 그 부분만 골라 삭제(forget_profile)하고, 그 외엔 평소처럼 정보를 저장하는 처리 흐름. 비밀번호 같은 민감정보가 메시지에 섞여도 결과가 어떻게 되는지 같이 확인.

05_summary_memory.py

profile 정보 4개 + RAG 관련 질문·답변 8쌍을 계속 쌓은 뒤, "전체 대화 글자 수" vs "최근 대화(window) 글자 수" vs "요약(summary) 글자 수"를 직접 세어 비교. 대화가 길어져도 요약은 거의 안 늘어난다는 걸 숫자로 보여줌.

06_session_persistence.py

MemoryStore 클래스로 student-a, student-b 두 사용자의 기억을 각각 따로 관리하다가 JSON 파일로 저장(save)하고, 프로그램을 다시 켠 것처럼 JSON에서 복원(load)해도 두 사람 정보가 안 섞이는지 확인.

07_openai_memory_chatbot.py

Profile + Summary + 최근 대화를 하나의 프롬프트 문자열로 합쳐(build_prompt) 실제 OpenAI 모델(gpt-4o-mini)에 넘기는 파일. --run 없이 실행하면 실제 API 호출 없이 "DRY RUN" 가짜 응답만 보여주고, --run을 붙이면 진짜 모델을 호출함.
```
