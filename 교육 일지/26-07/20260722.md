##### Chatbot Memory
- 이전 대화를 기억하는 방법을 배운다. 
- 챗봇으로 이전 대화에서 필요한 정보를 저장, 요약, 삭제하고 다음 답변의 Context로 사용
- **RAG**가 외부 문서를 찾는 구조라면, **Memory**는 사용자와 나눈 대화의 맥락을 저장.
- GPT가 Context가 좀 더 길다. Claude는 짧다.
- Cluade에서 작업한것을 Codex로 통채로 들고 올 수 있는 플러그인이 있음. 혹은 인수인계 파일을 만들어주면 집어넣어서 그대로 사용 가능

#### 실습 코드 이해해보기 (01_chatbot_memory.py)
```
1. clean_korean_suffix (4~9줄)
"민수야"처럼 문장 끝에 붙는 "야"/"이야"를 떼어내는 함수예요. "민수야" → "민수"

2. ChatMemory.__init__ (14~18줄)
메모리가 담을 3개의 바구니를 만듭니다.

messages: 전체 대화 기록
profile: 이름/관심사/목표 같은 사용자 정보
summary: 요약 문장

3. add_message (20~25줄)
대화 한 마디(역할 + 내용 + 시간)를 messages 리스트에 추가.

4. recent_messages (27~28줄)
messages[-6:]처럼 뒤에서 N개만 잘라서 반환 — 프롬프트가 무한정 길어지지 않게 "최근 것만" 쓰는 용도.

5. remember_profile (30~43줄) ← 어제 말씀드린 버그가 있는 곳
문장에 특정 키워드가 있으면 그 뒤 내용을 잘라서 profile에 저장합니다. 문제는 "관심사"만 검사해서, 정보를 알려줄 때("관심사는 RAG야")도 걸리고 질문할 때("관심사가 뭐였지?")도 걸려버려요.

6. update_summary (45~65줄)
profile에 쌓인 값들을 문장으로 이어붙여 요약을 만듭니다. (진짜 서비스라면 이 요약을 LLM이 만들지만, 여기서는 그냥 문자열 조합으로 흉내만 낸 것)

7. context (67~74줄)
profile + summary + recent_messages 3개를 한 묶음으로 반환 — 실제로 LLM에 보낼 프롬프트를 만들 때 이 묶음을 그대로 끼워 넣게 됩니다 (7번 파일에서 이렇게 씀).

8. generate_reply (76~100줄)
사용자 메시지를 보고 "이름을 물어보는 건가?", "요약해달라는 건가?"를 if문으로 패턴 매칭해서 답을 고릅니다. 진짜 LLM 없이 규칙만으로 흉내 낸 챗봇이에요.

9. chat_once (102~106줄)
저장 → 답변 생성 → 답변 저장, 한 사이클.
```
