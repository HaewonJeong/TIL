# 1부. Numpy 심화

## 1) 집계 함수 (`07_numpy_aggregation`)
| 함수 | 설명 |
|---|---|
| `.sum()` | 전체 요소 합계 (다차원도 전체 합) |
| `.mean()` | 평균 |
| `.max()` / `.min()` | 최댓값 / 최솟값 |
| `.argmax()` / `.argmin()` | 최댓값/최솟값의 **위치(index)** 반환 (값이 아님) |
| `axis=0` | 열 방향(세로) 집계 → 과목별 평균 등 |
| `axis=1` | 행 방향(가로) 집계 → 학생별 평균 등 |

```python
scores = np.array([80, 90, 100, 70])
scores.argmax()   # 2 (100의 위치)
scores.argmin()   # 3 (70의 위치)
```

## 2) 벡터 (`08_numpay_vector`)
- Numpy에서 **1차원 배열 = 벡터**
- **노름(norm, 벡터의 크기)**: `np.linalg.norm(v)` → 원점에서 벡터 끝까지의 거리 (피타고라스 정리 확장)
  ```python
  np.linalg.norm([3, 4])     # 5.0
  ```
- **내적(dot product)**: 같은 위치 요소끼리 곱한 뒤 모두 더한 값
  ```python
  a = np.array([1,2,3]); b = np.array([3,4,5])
  np.dot(a, b)   # 26
  a @ b          # 26 (동일한 결과, @ 연산자로도 가능)
  ```

## 3) 코사인 유사도 (`09_numpy_simiarity`)
- 공식: `cos(A,B) = (A·B) / (|A| × |B|)` → 내적을 두 벡터 크기의 곱으로 나눈 값
- **내적만으로는 한계가 있음**: 벡터 크기(길이)가 커지면 내적 값도 같이 커져서, "얼마나 비슷한 방향인지"를 그대로 비교하기 어려움
  ```python
  np.dot([1,1], [10,10])    # 20
  np.dot([1,1], [100,100])  # 200 → 방향은 같은데 값이 확 커짐
  ```
- **코사인 유사도는 크기(scale)에 영향받지 않고 방향(각도)만 비교**
  ```python
  cosine_similarity([1,1], [10,10])    # 1.0
  cosine_similarity([1,1], [100,100])  # 1.0 (동일)
  cosine_similarity([1,0], [0,1])      # 0.0 (수직 = 무관)
  cosine_similarity([1,0], [-1,0])     # -1.0 (반대 방향)
  ```
- 결과 범위: `-1`(정반대) ~ `0`(무관) ~ `1`(완전히 같은 방향) → **문장/단어 간 의미적 유사도 비교**에 핵심적으로 사용됨 (2부의 임베딩·어텐션과 직결)

# 2부. GPT & 프롬프트 엔지니어링

## 1) LLM(Large Language Model)이란
- 전 세계의 방대한 텍스트(지식)를 학습한 모델 → "무인도에 책만 잔뜩 들고 가 있는 것"에 비유
- 학습된 지식을 바탕으로 다음에 올 확률이 가장 높은 토큰을 예측해 문장을 생성

## 2) 트랜스포머(Transformer) 구조
- 구글 연구자가 발표한 논문에서 시작된 개념(해당 저자는 이후 Anthropic으로 이동)
- 핵심 구성: **인코더(Encoder)**와 **디코더(Decoder)**
  - **인코더**: 글을 **읽고 이해**하는 데 특화 (구글 계열 모델이 주력)
  - **디코더**: 새로운 결과를 **생성**하는 데 특화 (GPT 계열이 주력)
  - 인코더가 입력을 이해한 결과를 디코더가 받아 새로운 텍스트를 생성하는 구조
- 현재 대다수 LLM은 GPT(디코더 중심) 방식을 채택 — 방대한 파라미터(수십억 개)를 거쳐 결과를 생성

## 3) GPT의 핵심 처리 과정
1. **토큰화(Tokenization)**: 문장을 잘게 쪼갠 단위(토큰)로 분리 → 모델이 앞뒤 맥락을 더 잘 파악하도록 함
2. **벡터 임베딩(Embedding)**: 각 단어(토큰)의 의미를 다차원 공간상의 좌표(벡터)로 표현
3. **어텐션(Attention)**: 문맥 속에서 어떤 단어에 더 집중(가중치)할지 계산 → 그 가중치를 바탕으로 다음 토큰을 예측

## 4) 실습: 토큰화 → 임베딩 → 어텐션 (`01_token_embedding_attention`)

**① 토큰화**
```python
tokenize("나는 사과를 먹었다.")  # ['나는', '사과를', '먹었다.']
```

**② 임베딩 + 코사인 유사도로 단어 간 유사성 비교**
```python
embeddings = {
    "사과": [0.9, 0.1, 0.2],
    "바나나": [0.8, 0.1, 0.3],
    "자동차": [0.1, 0.9, 0.7],
}
# 사과 vs 바나나 → 유사도 높음 (같은 과일)
# 사과 vs 자동차 → 유사도 낮음 (관련성 적음)
```

**③ 어텐션(가중치 계산)**
- `query`(지금 찾고 싶은 정보) · `key`(각 토큰의 특징) · `value`(실제로 가져올 정보) 세 요소로 구성
- `scores = keys @ query` → 관련도 점수 계산
- `softmax(scores)` → 점수를 합이 1인 가중치(확률)로 변환
- `context = weights @ values` → 가중치만큼 value를 섞어 최종 결과 생성

**④ softmax로 다음 토큰 확률 예측**
```python
candidates = ["갔다", "먹었다", "달린다"]
scores = [1.2, 2.5, 0.3]
probabilities = softmax(scores)
best = candidates[np.argmax(probabilities)]  # 가장 확률 높은 다음 토큰
```

> 결론: GPT는 **토큰화 → 임베딩(벡터화) → 어텐션(가중치 계산) → 확률이 가장 높은 다음 토큰 선택**을 반복하며 문장을 만들어낸다.

## 5) 서비스 vs 모델의 차이
- **ChatGPT**: OpenAI가 만든 **서비스(웹/앱)** 이름
- **GPT-x 등 모델명**: 서비스 안에서 실제로 동작하는 **엔진(모델)**
- 개발자가 직접 자신의 서비스에 LLM을 붙이고 싶다면 `platform.openai.com`(Open AI API)에서 별도로 결제/API Key 발급 필요 (소액 결제로 샘플 테스트 가능)
- 앞으로는 토큰을 수동으로 나누는 대신 **API Key**를 이용해 실습 예정 (모델이 계속 최신화되므로 실습도 그때그때 함께 진행)
