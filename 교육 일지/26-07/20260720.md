## 1. PromptTemplate / ChatPromptTemplate

`01_prompt_template.py` 실습에서 두 가지 프롬프트 템플릿 방식을 다뤘다.

**PromptTemplate.from_template()**

문자열 템플릿 안에 `{audience}`, `{topic}` 같은 변수를 넣고, `invoke()`로 값을 채워 넣는 방식이다.

```python
template = PromptTemplate.from_template(
    """
    너는 AI 기초를 가르치는 강사다.
    {audience}에게 {topic}을 쉽게 설명해라.
    """
)
prompt_value = template.invoke({
    "audience": "Python과 Numpy를 막 배운 학생",
    "topic": "Embedding"
})
```

**ChatPromptTemplate.from_messages()**

system/user 역할을 나눠서 대화형 프롬프트를 구성하는 방식이다. 시스템 메시지로 AI의 역할을 지정하고, user 메시지에 질문과 답변 형식을 함께 넣었다.

```python
chat_prompt = ChatPromptTemplate.from_messages([
    ("system", "너는 초보자를 가르치는 AI 강사다. 어려운 용어는 쉽게 풀어쓴다."),
    ("user", "질문:{question}\n답변 형식: {answer_format}")
])
```

**포인트**: 프롬프트에 "전문 용어는 쉬운 말로 정의", "마지막에 한 줄 요약" 같은 조건을 명시하면 출력 품질이 좋아진다는 것을 확인했다.

## 2. LCEL로 체인 구성하기

`02_runnable_chain_basic.py`에서 LCEL의 핵심 패턴을 배웠다.

```
invoke() -> PromptTemplate -> ChatOpenAI -> StrOutputParser
```

프롬프트, 모델, 출력 파서를 `|` 파이프 연산자로 연결해서 하나의 체인으로 만든다.

```python
chain = template | model | StrOutputParser()
result = chain.invoke({
    "topic": "Attention",
    "audience": "Python과 Numpy를 막 배운 학생"
})
```

**포인트**:
- `.env` 파일에서 `OPENAI_API_KEY`를 불러오고, 키가 없으면 명확한 에러 메시지를 띄우도록 사전 검증 로직을 넣었다.
- `ChatOpenAI` 생성 시 `model`, `temperature`, `max_tokens`로 답변의 창의성과 길이를 조절할 수 있다.
- LCEL을 쓰면 프롬프트 → 모델 호출 → 후처리 과정을 코드 한 줄(`template | model | parser`)로 표현할 수 있어 가독성이 좋다.

## 3. Output Parser로 구조화된 출력 받기

`03_output_parser_practice.py`에서는 감정 분석 결과를 자유 텍스트가 아닌 **JSON 형식**으로 강제해서 받는 방법을 실습했다.

1. Pydantic `BaseModel`로 원하는 출력 스키마를 정의한다.
   ```python
   class SentimentAnalysis(BaseModel):
       label: str = Field(description="감정 분석 결과('긍정', '부정', '중립' 중 하나)")
       reason: str = Field(description="그렇게 판단한 상세 이유")
   ```
2. `JsonOutputParser(pydantic_object=...)`로 파서를 만들면, `parser.get_format_instructions()`가 모델에게 보여줄 포맷 지시문을 자동 생성해준다.
3. 이 지시문을 프롬프트의 `partial_variables`에 넣어서, 모델이 항상 정해진 JSON 구조로 답하도록 유도한다.
4. 체인은 `template | model | parser` 형태로 동일하게 구성하되, 마지막 파서가 문자열이 아니라 파싱된 딕셔너리를 반환한다.

**포인트**: `result["label"]`, `result["reason"]`처럼 결과를 바로 딕셔너리로 다룰 수 있어서, 후속 로직(조건 분기, DB 저장 등)에 연결하기 편하다는 것을 체감했다.

## 4. 실습: 퀴즈 생성 AI 프로그램

`04_quiz_generation.py`에서 지금까지 배운 요소(PromptTemplate + LCEL + ChatOpenAI + JsonOutputParser)를 종합해서 객관식 퀴즈 생성기를 만들었다.

- `Quiz` Pydantic 모델로 `question`, `choices`(4개 보기), `answer`, `explanation` 필드를 정의
- 프롬프트에 "당신은 AI 강사입니다"라는 역할과 주제(`{topic}`)를 넣고, `format_instructions`로 JSON 출력 형식을 강제
- `chain.invoke({"topic": "Embedding"})`로 실행하면 문제/보기/정답/해설이 구조화된 형태로 반환됨
- 반환된 `result["choices"]`를 `enumerate`로 순회하며 1~4번 보기로 출력

**느낀 점**: 이전 실습들에서 배운 개별 요소(템플릿, 체인, 파서)를 조합하면 "주제만 입력하면 퀴즈가 자동 생성되는" 실용적인 프로그램을 만들 수 있다는 것을 확인했다.

## 5. OpenAI Fine-tuning(파인튜닝)

### 5-1. Fine-tuning용 JSONL 데이터셋 (`fine_tuning_dataset_example.jsonl`)

- 한 줄에 하나의 대화 예제(JSON 객체)를 담는 JSONL 형식을 사용
- 각 줄은 `system`(교육 조교 역할 지정) / `user`(질문) / `assistant`(모범 답변) 메시지로 구성
- 예시 주제: NumPy의 `shape`, `ndim`, `dtype`, 인덱싱/슬라이싱, Broadcasting, axis, 평균/표준편차, 벡터 내적과 LLM, Attention, 프롬프트 엔지니어링, RAG, 파인튜닝, JSONL, training/validation 데이터, loss, overfitting, Function Calling, tool schema, Tools vs RAG 등 총 20개 대화 쌍
- 이 데이터셋의 마지막 예제(20번)는 "오늘 수업 내용을 3줄로 복습"하는 내용으로, 파인튜닝·JSONL·Function Calling을 핵심 요약해주는 답변이 포함됨

### 5-2. Fine-tuning Job 생성 코드 

```python
client = OpenAI()

with dataset.open("rb") as f:
    uploaded = client.files.create(file=f, purpose="fine-tune")

job = client.fine_tuning.jobs.create(
    training_file=uploaded.id,
    model="gpt-4.1-mini"
)
```
