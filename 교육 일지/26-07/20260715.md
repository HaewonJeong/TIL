## 1. os 모듈

운영체제와 상호작용하는 기능을 제공하는 표준 라이브러리. 디렉터리/파일 처리, 경로 처리, 환경변수 접근 등을 담당한다.

### 1) 디렉터리 관련 기능
| 기능 | 설명 |
|---|---|
| `os.getcwd()` | 현재 작업 디렉토리 반환 |
| `os.chdir(path)` | 작업 디렉토리 변경 |
| `os.listdir(path='.')` | 디렉토리 내 파일/폴더 목록 반환 |
| `os.mkdir(path)` | 디렉토리 1개 생성 |
| `os.makedirs(path)` | 중간 경로 포함 여러 디렉토리 한 번에 생성 |
| `os.rmdir(path)` | 디렉토리 삭제 (비어있어야 함) |
| `os.removedirs(path)` | 중간 경로 포함 여러 디렉토리 삭제 |

### 2) 파일 관련 기능
| 기능 | 설명 |
|---|---|
| `os.remove(path)` | 파일 삭제 |
| `os.rename(src, dst)` | 파일/디렉토리 이름 변경 |
| `os.stat(path)` | 파일의 메타정보 조회 |

### 3) 경로 관련 기능 (`os.path`)
| 기능 | 설명 |
|---|---|
| `os.path.join(a, b)` | 경로 합치기 (OS에 맞는 구분자 자동 처리) |
| `os.path.exists(path)` | 존재 여부 확인 |
| `os.path.isfile(path)` | 파일 여부 확인 |
| `os.path.isdir(path)` | 디렉토리 여부 확인 |
| `os.path.abspath(path)` | 절대 경로로 변환 |
| `os.path.basename(path)` | 파일명만 추출 |
| `os.path.dirname(path)` | 디렉토리 경로만 추출 |
| `os.path.splitext(path)` | 파일명/확장자 분리 |

### 4) 환경변수 & 실행
- `os.environ`: 환경변수 딕셔너리 (`os.environ['PATH']`)
- `os.getenv('KEY')`: 환경변수 조회
- `os.system(cmd)`: 터미널 명령 실행

### 5) 실습 코드
```python
import os

print(os.getcwd())          # 현재 경로 확인
print(os.listdir('./'))     # 현재 폴더 목록
print(os.listdir('../'))    # 상위 폴더 목록

print(os.path.isfile('./01_os.ipynb'))  # True

# 폴더가 없을 때만 생성 (중복 생성 방지 패턴)
if not os.path.exists('test'):
    os.mkdir('test')
    print('test 디렉토리를 생성했습니다.')
else:
    print('test 디렉토리가 존재합니다.')

# os.path.join()으로 안전하게 경로 조합 후 파일 쓰기/읽기
file_path = os.path.join('test', 'helloworld.txt')
with open(file_path, 'w') as f:
    f.write('Hello world')

with open(file_path, 'r') as f:
    print(f.read())
```

## 2. 파일 입출력 IO 
### `open()` 함수 기본 문법
```python
f = open('파일경로', '모드', encoding='인코딩')
```

### 주요 모드
| 모드 | 의미 | 파일이 없을 때 |
|---|---|---|
| `'r'` | 읽기 | `FileNotFoundError` 발생 |
| `'w'` | 쓰기(덮어쓰기) | 새로 생성 |
| `'a'` | 추가(이어쓰기) | 새로 생성 |
| `'x'` | 배타적 생성 | 이미 존재하면 오류 |
| `'b'` | 바이너리 모드 | 이진 데이터 처리용 |
| `'t'` | 텍스트 모드(기본) | 생략 가능 (`'rt'` = `'r'`) |

> 예: `'rb'`(이진 읽기), `'wt'`(텍스트 쓰기)

### 읽기/쓰기 관련 메서드
- `read()`: 전체 내용을 문자열로 읽기
- `readline()`: 한 줄씩 읽기
- `readlines()`: 모든 줄을 리스트로 반환
- `write()`: 문자열 1개 작성
- `writelines()`: 문자열 리스트를 한 번에 작성

> `01_os` 실습에서 `with open(...) as f:` 구문으로 파일을 열고, 블록이 끝나면 자동으로 `close()`되는 방식(컨텍스트 매니저)까지 함께 확인함

## 3. 예외처리 Exception 

### Exception vs Error
- **Exception**: 정상 흐름으로 복구 가능한 미약한 오류 → `try-except`로 복구
- **Error**: 복구 불가능한 심각한 오류 → 재시작, 코드 수정 필요
- 파이썬은 Exception과 Error를 엄격히 구분하지 않으며, `TypeError`, `ValueError`, `MemoryError` 등도 모두 `Exception`의 하위 클래스

### 예외 계층 구조
```
BaseException
├── SystemExit          # 프로그램 종료
├── KeyboardInterrupt   # Ctrl+C
├── GeneratorExit       # 제너레이터 종료
└── Exception           # 대부분의 예외가 여기 속함
    ├── TypeError
    ├── ValueError
    ├── MemoryError
    ├── OSError
    └── ...
```

### try-except 기본 구조
```python
try:
    print(10 / 0)
except Exception as e:   # 발생한 예외 객체
    print(e)              # division by zero

print('프로그램 정상 운영중')  # except 이후 정상 흐름 이어짐
```

### 여러 타입의 예외 처리 (예외별 분기)
```python
try:
    n = random.randint(0, 2)
    if n == 0:
        print(10 / 0)
    elif n == 1:
        int('ㅋㅋㅋㅋㅋ')
    else:
        lst = [1, 2, 3]
        print(lst[100])
except ZeroDivisionError as e:
    print('0으로 나누시면 안됩니다.')
except IndexError as e:
    print('존재하지 않는 인덱스를 참조하시면 안됩니다.')
except Exception as e:
    print('오류가 발생했습니다. :', e)
```

### try - except - else - finally 전체 구조
- **try**: 실행할 코드
- **except**: 예외 발생 시 실행
- **else** (선택): try 블록이 예외 없이 끝났을 때만 실행
- **finally** (선택): 예외 발생 여부와 상관없이 항상 실행
```python
import traceback

try:
    f = open('hello2.txt', 'r', encoding='utf-8')
except Exception as e:
    print(e)
    traceback.print_exc()   # 상세 오류 추적 출력
else:
    print('내용: ', f.read())
    f.close()
finally:
    print('무조건 실행')
```

## 4. Numpy 기초

### 1) Numpy 소개 및 List와의 차이
```python
import numpy as np
print(np.__version__)   # 설치 버전 확인
```
- **List의 `+`**: 리스트 연결(concatenation) → `[1,2,3] + [4,5,6]` = `[1,2,3,4,5,6]`
- **Numpy 배열의 `+`**: 같은 위치 요소끼리 계산(element-wise) → `[1,2,3] + [4,5,6]` = `[5,7,9]`

### 2) 엑셀처럼 2차원 데이터 다루기
```python
scores = np.array([
    [80, 90, 70],
    [95, 85, 88],
    [70, 78, 84]
])
scores.mean()          # 전체 평균
scores.max()           # 최고점
scores.min()            # 최저점
scores.mean(axis=0)    # 과목별(열 방향) 평균
scores.mean(axis=1)    # 학생별(행 방향) 평균
```
> `axis=0`은 열 방향(위→아래), `axis=1`은 행 방향(왼쪽→오른쪽) 집계 기준

### 3) 배열 생성 함수 6가지
| 함수 | 설명 |
|---|---|
| `np.array()` | 리스트 → 배열 변환 |
| `np.zeros(shape)` | 0으로 채운 배열 생성 |
| `np.ones(shape)` | 1로 채운 배열 생성 |
| `np.full(shape, value)` | 지정한 값으로 채운 배열 생성 |
| `np.arange(start, end, step)` | 일정한 **간격**으로 생성 |
| `np.linspace(start, end, num)` | 일정한 **개수**로 균등 분할 생성 |

```python
np.array([90, 80, 100])                 # 1차원 배열
np.array([[1,2,3],[4,5,6]])              # 2차원 배열 (행 길이는 모두 같아야 함)
np.array([1,2,3], dtype=float)           # dtype 명시 가능

np.zeros(5)          # [0. 0. 0. 0. 0.] → 기본 dtype은 float
np.zeros((3,4))      # 2차원(행,열) 0 배열

np.ones(5)
np.ones((2,4))

np.full((3,4), 100)  # 100으로 채운 (3,4) 배열

np.arange(10)          # 0~9
np.arange(5, 20, 3)    # 5,8,11,14,17 (step 기준)

np.linspace(0, 1, 5)   # 0~1을 5개로 균등 분할 (개수 기준)
np.linspace(0, 10, 6)
```
> **`arange` vs `linspace`**: `arange`는 **간격(step)** 기준, `linspace`는 **개수(num)** 기준으로 값을 생성한다는 차이를 비교 실습

### 4) 배열의 속성 (attributes)
| 속성 | 설명 |
|---|---|
| `.shape` | 배열의 모양을 튜플로 반환 (1차원: `(개수,)`, 2차원: `(행,열)`, 3차원: `(깊이,행,열)`) |
| `.ndim` | 배열의 차원 수 |
| `.size` | 전체 요소 개수 (shape 값들의 곱) |
| `.dtype` | 배열 요소의 자료형 |

```python
np.array([80, 90, 100]).shape              # (3,)
np.array([[80,90,100],[70,88,99]]).shape   # (2, 3)
np.zeros((2,3,4)).shape                    # (2, 3, 4)

np.array([80, 90, 100]).ndim               # 1
np.zeros((2,3,4)).ndim                     # 3

np.array([[80,90,100],[99,24,67]]).size    # 6
np.zeros((2,3,4)).size                     # 24

np.array([10, 20, 30]).dtype               # int
np.array([10., 20., 30.]).dtype            # float

# Numpy 배열은 하나의 자료형만 저장 가능 -> 섞이면 자동으로 통일된 타입(문자열 등)으로 변환됨
data = np.array([100, 200, "Python"])
print(data, data.dtype)  # 모두 문자열(str)로 변환됨
```
