# Spring-data-JPA 코드
#### Model Mapper
- 객체와 객체 사이의 데이터를 자동으로 복사(매핑)해주는 라이브러리
- 동일한 이름의 필드를 기준으로 객체 간 데이터를 복사 가능
#### JpaRepository
- Menu 엔티티를 대상으로 CRUD 기능을 제공하는 Spring Data JPA 인터페이스
- 원래 JPA는 아래 코드 직접 작성 
```
@PersistenceContext  
private EntityManager entityManager;

entityManager.persist(menu);  
entityManager.find(Menu.class, id);  
entityManager.remove(menu);
```

- 그런데 Spring이 알아서 구현체를 만들어줌. 
- 첫번째 제내릭 : 어떤 어떤 엔티티를 관리할 것인가?
- 두번째 제네릭 : PK 타입 의미
```
public interface MenuRepository  
extends JpaRepository<Menu, Integer> {  
}
```
- 이렇게 하면 아무것도 안써도
```
menuRepository.findAll();
```

- 전체 조회

---

```
menuRepository.findById(1);
```

- PK 조회

---

```
menuRepository.save(menu);
```

- 등록/수정

---

```
menuRepository.delete(menu);
```

- **실제 실무에서 쓰이는 방식**
	- 쇼핑몰 상품 목록, 게시판 목록, 회원 목록, 주문 내역 조회 등 대량의 데이터를 조회하는 화면에서 사용된다.
	- 모든 데이터를 한 번에 조회하지 않고 필요한 개수만 조회하여 데이터베이스 부하와 네트워크 트래픽을 줄일 수 있다.
	- 프론트엔드는 `totalPages`, `first`, `last` 정보를 이용하여 페이지 이동 버튼을 구현한다.
	- 예를 들어 10만 건의 게시글이 있는 게시판에서 한 페이지당 20개씩 조회하는 방식으로 사용된다.
