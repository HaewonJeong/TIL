# 🖥️ GitHub Issue 관리 가이드

> 냉파마스터 프로젝트 이슈 관리 가이드

---

## 1. GitHub Issue 등록 방법

### 이슈 제목 형식

```
[기능] 작업 내용
```

**예시:**
```
[냉장고] 냉장고 및 재료 관리 기능 구현
```

### 주요 도메인별 이슈 분류

| 도메인 | 이슈명 |
|---|---|
| 회원 | [회원] 회원 기능 구현 |
| 레시피 | [레시피] 레시피 및 추천 기능 구현 |
| 냉장고 | [냉장고] 냉장고 및 재료 관리 기능 구현 |
| 냉파점수 | [냉파점수] 냉파점수 및 통계 기능 구현 |
| 관리자 | [관리자] 관리자 기능 구현 |
| 정보/문의 | [정보/문의] 정보글 및 문의 기능 구현 |
| 공통 | [공통] API 연동 및 QA |
| 배포 | [배포] 배포 및 운영 환경 설정 |

## 2. 이슈 본문 템플릿

### feat / refactor / docs

```markdown
## 작업 목표
- 구현할 내용을 bullet point로 작성

## 완료 조건 (Definition of Done)
- [ ] 체크리스트 형식으로 완료 기준 명시
- [ ] 예: 재료 등록 API 응답 200 확인
- [ ] 예: 예외 처리 (중복 재료, 유효하지 않은 날짜)

## 참고 문서
- 선택 기재
- API 명세서, ERD, 기획서 링크 등
```

### bug

```markdown
## 버그 설명
어떤 문제가 발생했는지 간략히 설명

## 재현 방법
1.
2.

## 기대 결과
어떻게 동작해야 하는지

## 실제 결과
현재 어떻게 동작하는지

## 환경
- BE / FE 구분
- 관련 브랜치:
```

### discuss

```markdown
## 논의 주제
- 왜 이 결정이 필요한지

## 선택지
- A안: ...
- B안: ...

## 결정 기한
- YYYY-MM-DD까지

## 참여자
- @팀원 태그
```

## 3. 이슈 라벨 기준

| 라벨 | 색상 | 의미 |
|---|---|---|
| `feat` | 🟢 초록 | 새 기능 구현 |
| `bug` | 🔴 빨강 | 버그 수정 |
| `refactor` | 🔵 파랑 | 코드 개선 |
| `docs` | 🟡 노랑 | 문서 작업 |
| `discuss` | 🟣 보라 | 논의/결정 필요 |
| `blocked` | ⚫ 검정 | 진행 불가 상태 |
| `BE` | 🟠 주황 | 백엔드 작업 |
| `FE` | 🩵 하늘 | 프론트엔드 작업 |

---

## 4. 칸반보드 운영 규칙

### 컬럼 정의

| 컬럼 | 이동 시점 |
|---|---|
| `Todo` | 이번 스프린트에 하기로 확정된 작업 |
| `In Progress` | 지금 실제로 작업 중 |
| `Done` | PR 머지 + 기능 확인까지 완료 |
| `Blocked` | 다른 작업, 협업, API/DB 의존으로 막힘 |

---

## 5. 이슈 작성 시 주의사항

- 이슈 제목만 보고 내용을 파악할 수 있어야 한다
- 버그 이슈는 **재현 방법**을 반드시 포함한다
- 기획 변경이 발생하면 관련 이슈와 문서를 **함께** 수정한다
- 해결된 이슈는 **완료일과 처리 내용**을 코멘트로 남기고 닫는다
- 이슈를 열어두고 방치하지 않는다 — 진행 불가 시 `Blocked` 처리
