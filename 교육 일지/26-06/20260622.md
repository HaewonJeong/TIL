## 2. Spring Security

### 인증과 인가
- **인가(Authorization)**: 로그인 시 User/Admin 등 권한 구분
- **인증(Authentication)**: 사용자가 누구인지 확인 (JWT 토큰 사용 시 필요)

### 과거 vs 현재
- Servlet 사용 시에는 Security를 한 땀 한 땀 직접 구현해야 했음 → Spring Security가 이를 추상화

### 세션 기반 인증
```
서버 (Session 정보) → User (Session ID)
User → Load Balancer → A/B/C 서버
```
- 문제점: B 서버 장애 발생 → 사용자가 A서버로 이동 → 세션 정보가 없어 "로그인 하세요" 발생
- 해결: 세션 정보를 서버 간 동기화/공유해야 함 (Redis 등 활용)

### 토큰 기반 인증 (JWT)
```
User → 토큰(JWT) → LB → A/B/C → DB
```
- 사용자가 요청 시 JWT 토큰을 들고 감
- 서버(A/B/C 중 어디든)는 토큰(이용권/키)이 유효한지만 검증 → 통과
- 서버 간 세션 동기화 문제가 없어 확장성이 좋음

### 렌더링 방식
- **SSR (Server Side Rendering)**: 서버에서 화면을 그려서 전달
- **CSR (Client Side Rendering)**: 클라이언트(브라우저)에서 화면을 그림

### 흐름 예시
```
User → index.html → 서버
서버 ← id/pwd (login) ← index.html
```

- JSP → Servlet → `org.springframework` 핸들러 메서드로 발전된 흐름

## 3. Filter와 Interceptor

| 구분 | 설명 |
|---|---|
| Filter | HTTP 요청을 받아주는 것. 서버(컨테이너)에서 관리 |
| Interceptor | Filter와 달리 Spring Bean 객체에 접근 가능. 에러 처리 등에 주로 사용 |

- **Authentication Filter**: 인증 처리를 담당하는 필터
- React ↔ Spring Boot 구조에서 인증/인가 처리, 포워드 방식과 리다이렉트 방식의 차이 학습

## 5. MySQL 연결 설정

### 계정 구분
- `mysql@localhost`: 로컬 개발용
- `mysql@naver.com`: 실제 운영 서버용

### 계정 생성 명령어

```sql
-- 계정 생성
create user 'spcamp'@'%' identified by 'spcamp';

-- 데이터베이스 생성
create database cqrs;

-- 권한 부여
grant all privileges on cqrs.* to 'spcamp'@'%';
```

### application.yml 설정 예시

```yaml
spring:
  application:
    name: springsecurity
  datasource:
    driver-class-name: com.mysql.cj.jdbc.Driver
    url: jdbc:mysql://localhost:3306/cqrs
    username: spcamp
    password: spcamp
  jpa:
    hibernate:
      ddl-auto: create
    show-sql: true
    properties:
      hibernate:
        format_sql: true
```

## 7. 프로젝트 설계 - 모놀리식 vs MSA

- 프로젝트 진행 시 **도메인 단위로 나누는 것**이 중요

| 구분 | 설명 |
|---|---|
| 모놀리식 (Monolithic) | 모든 도메인이 하나의 서버에 같이 존재. 결제 도메인에 장애 발생 시 전체 서버가 영향을 받을 수 있음. 서버 확장 시 모든 도메인을 함께 복제해야 함 |
| MSA (Microservice Architecture) | 도메인별로 서버를 독립적으로 분리. 한 도메인의 장애가 다른 도메인에 영향을 주지 않음 |
