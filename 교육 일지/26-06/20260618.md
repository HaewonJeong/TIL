## REST API

### 1. 목표
- Spring Boot에서 REST API를 구현하기 전에, API를 어떻게 설계해야 하는지 정리

### 2. REST API란 무엇인가
- 요청 상태 코드
- 기능의 이름보다 데이터의 대상, 리소스를 표현하는 것이 좋다.

### 3. URL은 동사가 아니라 리소스 중심으로 작성한다
- URL = 대상, 자원
- HTTP Method = 행위

### 4. HTTP Method 역할
- PUT : 전체 수정에 가깝고, PATCH는 일부 수정에 가깝다.
- 팀 플젝에서는 일관되게 정해놓고 가는 것이 좋다.

### 6. 단수보다 복수형 리소스명을 사용한다
- REST API에서는 보통 리소스 이름을 복수형으로 작성

### 7. Path Variable과 Query String 구분
- **Path Variable** : 특정 리소스를 식별할 때 사용
- **Query String** : 검색, 필터, 정렬, 페이징처럼 조회 조건을 표현할 때 사용

### 8. 중첩 리소스 설계
- 리소스 사이에 부모-자식 관계가 있을 때는 URL을 계층적으로 표현할 수 있다.

### 9. Request Body 설계
- POST, PUT, PATCH 요청에서는 보통 JSON body를 사용한다.

### 10. Response Body 설계
- 응답 body에는 클라이언트가 화면을 그리는 데 필요한 데이터를 담는다.

### 11. HTTP 상태 코드
REST API에서는 응답 body뿐 아니라 HTTP 상태 코드도 중요하다.

| 상태 코드 | 의미 | 사용 예시 |
|---|---|---|
| 200 OK | 요청 성공 | 조회 성공 |
| 201 Created | 생성 성공 | 회원가입, 게시글 등록 |
| 204 No Content | 성공했지만 응답 body 없음 | 수정, 삭제 성공 |
| 400 Bad Request | 잘못된 요청 | 검증 실패 |
| 401 Unauthorized | 인증 실패 | 로그인하지 않음 |
| 403 Forbidden | 권한 없음 | 로그인했지만 접근 권한 없음 |
| 404 Not Found | 리소스 없음 | 없는 게시글 조회 |
| 409 Conflict | 충돌 | 이메일 중복 |
| 500 Internal Server Error | 서버 오류 | 예상하지 못한 오류 |

### 12. 401과 403 차이
- **401 Unauthorized** : 인증되지 않은 사용자다.
- **403 Forbidden** : 인증은 되었지만 권한이 없다.

### 15. REST API 설계 실습

**검증 실패 예시**

```json
{
  "status": 400,
  "errorCode": "VALIDATION_FAILED",
  "message": "요청 값 검증에 실패했습니다.",
  "errors": {
    "title": "제목은 필수입니다.",
    "content": "내용은 필수입니다."
  }
}
```

**리소스 없음 예시**

```json
{
  "status": 404,
  "errorCode": "POST_NOT_FOUND",
  "message": "게시글을 찾을 수 없습니다."
}
```

**중복 예시**

```json
{
  "status": 409,
  "errorCode": "EMAIL_DUPLICATED",
  "message": "이미 사용 중인 이메일입니다."
}
```

### 16. 팀 프로젝트 API 명세 작성 규칙
- 팀 프로젝트에서는 API 명세를 반드시 작성해야 한다.
- React와 Spring이 동시에 개발되기 때문에, API 약속이 없으면 화면과 서버가 맞지 않는다.
- API 명세에는 최소한 다음 항목이 필요하다.

### 17. API 명세 예시
- 게시글 등록

### 18. 화면-API 매핑 예시

| 화면 | 사용자 액션 | 호출 API | 성공 처리 | 실패 처리 |
|---|---|---|---|---|
| 로그인 화면 | 로그인 버튼 클릭 | `POST /api/v1/auth/login` | 메인 화면 이동 | 에러 메시지 표시 |
| 게시글 목록 | 화면 진입 | `GET /api/v1/posts` | 목록 렌더링 | 빈 목록 또는 에러 표시 |
| 게시글 상세 | 화면 진입 | `GET /api/v1/posts/{postId}` | 상세 정보 표시 | 404 페이지 또는 목록 이동 |
| 게시글 등록 | 등록 버튼 클릭 | `POST /api/v1/posts` | 상세 페이지 이동 | 입력값 오류 표시 |
| 마이페이지 | 화면 진입 | `GET /api/v1/members/me` | 내 정보 표시 | 로그인 페이지 이동 |

### 19. 정리
REST API 설계에서 가장 중요한 것은 다음과 같다.

1. URL은 리소스 중심으로 작성한다.
2. 행위는 HTTP Method로 표현한다.
3. 조회 조건은 Query String으로 표현한다.
4. 특정 리소스 식별자는 Path Variable로 표현한다.
5. 요청과 응답 JSON 구조를 명확히 정한다.
6. 상황에 맞는 HTTP 상태 코드를 사용한다.
7. React와 Spring이 함께 개발하려면 API 명세가 반드시 필요하다.
