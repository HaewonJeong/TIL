# MyBatis 동적 쿼리
### if
조건에 따라 SQL 일부를 추가

```
<if test="name != null">
```

---
### choose
Java switch와 유사
```
<choose>    <when>    <otherwise></choose>
```
---
### foreach
반복문
```
<foreach collection="list">
```
주로 IN 절 생성
```
WHERE menu_code IN (1,2,3,4,5)
```
---
### trim
동적으로 생성된 AND, OR 제거

```
<trim prefix="WHERE"      prefixOverrides="AND|OR">
```
---

### 오늘 겪은 오류
- mapper namespace와 인터페이스 패키지명 일치해야 함
- select id와 메서드명 일치해야 함

# 수업중 메모
- choose 태그 : if는 모두 동작, choose 하나만 동작  
    - when과 otherwise와 함께 쓸 수 있다.                             
    - when : 조건에 부합하면 쿼리에 포함                             
    - otherwise : 그렇지 않을때 쿼리에 포함
- where 태그
	- 엘리먼트 구문이 where로 시작하지 않으면 자동으로 where을 붙여 준다.
	- 또한, 엘리먼트 내부에 모든 쿼리문이 추가되지 않는 상황인 경우 where를 무시한다.
	- 엘리먼트 구문이 AND나 OR로 시작하면 자동으로 해당 단어를 지워준다.
- trim 태그  
	- 조회 조건에 AND 혹은 OR로 시작하지 않는 다른 문자로 시작하는 경우 등, where 엘리먼트가 기본적으로 처리하는 기능에 추가 규칙을 정의하기 위해 trim 엘리먼트를 제공한다.

#### MyBatis 로그 출력 방법
	- <configuration> 하위에 <settings>를 추가하고 <setting>을 사용해서 로그 출력기를 지정한다.
```	
<settings>  
    <setting name="logImpl" value="STDOUT_LOGGING"/>  
</settings>
```
#### Java key와 xml의 collection이 연결된다.
- MyBatis의 <foreach>는 Java에서 전달한 컬렉션 이름을 사용한다.
- 예를 들어 Java 코드에서
```
Map<String, List<Integer>> criteria = new HashMap<>();
criteria.put("randomMenuCodeList", randomMenuCodeList);

mapper.searchMenuByRandomMenuCode(criteria);
```
- XML에서는
```
<foreach collection="randomMenuCodeList"
         item="menuCode"
         open="("
         separator=","
         close=")">
    #{menuCode}
</foreach>
```
- 처럼 사용한다. 즉,
```
criteria.put("randomMenuCodeList", ...)
```
- 의 key 이름과
```
collection="randomMenuCodeList"
```
- 가 정확히 일치해야 한다. 일치하지 않으면 오류가 발생한다.
```
The expression 'randomMenuCodeList' evaluated to a null value.
```

#### SqlSession 없이 MyBatis를 사용할 수 있다
- sql builder를 사용하지 않고 Mybatis 의존성을 build.gradle에 추가하면 도와준다. 
- implementation 'org.mybatis.spring.boot:mybatis-spring-boot-starter:4.0.1'
#### yml(야멀) 파일
- 원래 설정 파일은 .properties를 많이 썼는는데, 너무 길어져서 요즘은 application.yml을 많이 쓴다.
- @Mapper을 붙이면 스프링이 MenuMapper를 Bean으로 등록해줌.
- 예전방식
```
SqlSession session = getSqlSession();MenuMapper mapper = session.getMapper(MenuMapper.class);
```
#### foreach는 Java 컬렉션(List, Set, 배열)을 반복하여 SQL의 IN 절이나 다중 INSERT 문을 동적으로 생성하는 태그이다.
### 어노테이션 : @Mapper
```
@Mapperpublic interface MenuMapper

↓

@RequiredArgsConstructor@Servicepublic class MenuService {    private final MenuMapper menuMapper;}
```

#### 어노테이션 : @CrossOrigin(origins = "*")
```
1. **뜻**: 다른 주소(Origin)의 웹 페이지가 내 API를 호출하는 것을 허용한다.
2. **언제 사용?**: React(Vue) 프론트(`localhost:3000`)와 Spring Boot 백엔드(`localhost:8080`)를 따로 실행할 때.
3. **예시**: React에서 `fetch("http://localhost:8080/api/members")` 호출 시 CORS 에러가 나면 `@CrossOrigin`을 추가한다.
