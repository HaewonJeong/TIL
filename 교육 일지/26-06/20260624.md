# Spring Security 권한 관리 및 프로젝트 환경 설정 실습

## 항목 : 메서드 단위 권한 제어 (@PreAuthorize)

### 실습 내용

* Spring Security의 `@PreAuthorize` 어노테이션을 이용하여 메서드 실행 전에 권한을 검사하도록 설정하였다.
* 관리자(Admin) 권한을 가진 사용자만 사용자 목록 조회 API를 호출할 수 있도록 구현하였다.

```java
@PreAuthorize("hasAuthority('ADMIN')")
@GetMapping("/admin/users")
```

### 학습 내용

* `@PreAuthorize` : 메서드 실행 전에 권한 검사를 수행한다.
* `@PostAuthorize` : 메서드 실행 후 반환값을 기준으로 권한 검사를 수행한다.
* URL 기반 권한 제어보다 세밀한 접근 제어가 가능하다.

### 실제 실무에서 쓰이는 방식

* 관리자 전용 기능
* 회원 정보 조회
* 주문 취소 권한
* 게시글 수정 권한

등의 비즈니스 로직 단위 보안 처리에 활용된다.

### 실습 내용

* Spring Boot 설정 파일을 환경별로 분리하여 관리하였다.

```text
src/main/resources
 ├─ application.yml
 ├─ application-dev.yml
 ├─ application-prod.yml
```

### 학습 내용

* 개발 환경(dev)과 운영 환경(prod)의 설정을 분리할 수 있다.
* DB 정보, 로그 설정, API Key 등을 환경별로 다르게 관리할 수 있다.

### 실제 실무에서 쓰이는 방식

* 개발 서버
* 테스트 서버
* 운영 서버

별로 별도의 설정 파일을 사용하여 배포 환경을 관리한다.

## 항목 : 초기 데이터 관리 (initdb)

### 실습 내용

* 애플리케이션 실행 시 테스트용 데이터를 자동으로 생성하는 초기화 스크립트를 학습하였다.

### 학습 내용

* 개발 환경에서는 반복 테스트를 위해 초기 데이터를 자동 생성한다.
* 운영 환경에서는 사용하지 않는다.

### 실제 실무에서 쓰이는 방식

* 공통 코드 데이터
* 관리자 계정
* 테스트 데이터

초기 세팅을 자동화하기 위해 사용한다.

* Service 계층에서 로그를 남기는 방법에 대해 학습하였다.

```java
public class ReviewService {

    public void createReview() {

    }
}
```

* 예외 발생 기록
* 사용자 요청 추적

등을 위해 Service 계층에 로그를 작성한다.
