## React 비동기 통신 및 API 연동 실습 일지

### 1. 비동기 통신의 기초 (Fetch API)
- `fetch`는 브라우저 내장 API로, 서버에 데이터를 요청하고 응답을 받는 가장 기본적인 방법입니다.
- `fetch`는 페이지 전체를 새로고침하지 않고 필요한 부분만 데이터를 가져오는 Ajax 기술의 현대적 구현체입니다.

### 1-1. Promise와 JSON 파싱
- `fetch` 호출 시 즉시 데이터가 오는 것이 아니라 **Promise(미래의 응답)** 객체가 반환됩니다.
- 응답 본문을 읽으려면 `await response.json()`을 통해 자바스크립트 객체로 변환하는 과정이 필요합니다.

```
async function callAPI() {
    // 1. 서버에 요청 (Promise 반환)
    const response = await fetch('[https://jsonplaceholder.typicode.com/users](https://jsonplaceholder.typicode.com/users)');
    
    // 2. 응답 본문을 JSON으로 파싱
    const data = await response.json();
    console.log(data);
}
```
### 1-2. 응답 객체의 특징
- **`bodyUsed`**: 응답 본문은 스트림 형태이므로 한 번만 읽을 수 있습니다.
- **`response.ok`**: `fetch`는 404나 500 에러가 발생해도 통신 자체는 성공한 것으로 간주하여 `catch`로 가지 않습니다. 따라서 반드시 `if (!response.ok)`를 통한 수동 에러 처리가 필요합니다.

### 2. 비동기 처리 문법 비교
- 비동기 코드를 제어하는 두 가지 주요 방식
#### 2-1. .then() 체이닝
- 콜백 함수를 연결하여 순차적으로 실행하는 방식
- 성공(then)과 실패(catch)를 명확히 구분하여 처리
- 여러 비동기 작업을 병렬로 호출하거나 읽는 흐름을 연습할 때 유용하지만, 단계가 많아지면 관리가 복잡해질 수 있습니다.

#### 2-2. async / await (권장)
- 비동기 코드를 마치 동기 코드(위에서 아래로)처럼 읽히게 작성할 수 있어 가독성이 뛰어남.
- **`try-catch-finally`** 블록을 사용하여 에러 처리와 후속 작업을 깔끔하게 분리할 수 있습니다.

### 3. Axios 라이브러리 : fetch의 진화
- 외부 라이브러리인 Axios는 fetch의 번거로운 점들을 자동화해 줍니다.
#### 3-1. Axios의 주요 장점
| 특징 | Fetch API | Axios |
| :--- | :--- | :--- |
| **성격** | 브라우저 내장 기능 | 외부 라이브러리 (CDN/설치 필요) |
| **JSON 파싱** | `response.json()` 직접 호출 | **자동 파싱** (`res.data`로 즉시 접근) |
| **에러 처리** | 404/500 에러도 직접 검사 필요 | **자동 Reject** (404/500 시 catch로 이동) |
| **메서드** | 옵션 객체 설정 방식 | `.get()`, `.post()` 등 직관적 메서드 |

### 4. React 컴포넌트 데이터 바인딩
- 실제 React 컴포넌트 내에서 데이터를 가져와 화면에 그리는 실무 패턴을 익혔습니다.

### 4-1. 3가지 핵심 상태 관리 전
안정적인 화면 전환을 위해 로딩, 데이터, 에러 상태를 각각 관리합니다.
1. **`loading`**: 데이터가 오기 전까지 "로딩 중..." 메시지 표시.
2. **`data`**: 서버에서 성공적으로 가져온 데이터 리스트.
3. **`error`**: 통신 실패 시 사용자에게 보여줄 에러 객체.

### 4-2. useEffect를 활용한 데이터 호출
- 컴포넌트가 처음 나타날 때(Mount) 딱 한 번만 데이터를 가져오도록 의존성 배열([])을 설정합니다.

### 4-3. 실무형 방어적 코드 (Defensive Coding)
버그를 줄이고 렌더링 에러를 방지하기 위한 핵심 기법입니다.
- **Early Return**: `if (loading)`이나 `if (error)` 상태일 때 컴포넌트를 미리 반환하여 메인 로직의 복잡도를 줄입니다.
- **옵셔널 체이닝 (`?.`)**: `user.company?.name` 처럼 데이터가 깊은 구조일 때 부모 값이 없어도 에러를 내지 않고 `undefined`를 반환하게 합니다.
```
<!-- 회원 목록 출력 예제
 API에서 받은 내용 화면 출력 예제-->
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document</title>
    <!-- CDN -->
    <script crossorigin src="https://unpkg.com/react@18/umd/react.development.js"></script>
    <script crossorigin src="https://unpkg.com/react-dom@18/umd/react-dom.development.js"></script> 
    <script crossorigin src="https://unpkg.com/@babel/standalone/babel.min.js"></script>

    <style>
        .item{
            background-color: bisque;
            border: 1px dotted black;
            width: 350px;
            display: inline-block;
            margin: 10px;
        }
    </style>

</head>
<body>
    <div id="root"></div>

    <script type="text/babel"> 
        const { useEffect, useState } = React;

        //가장 간단한 dump component. 분리해두면 재사용에 용이
        function Title(){
            return <h1>회원 목록</h1>
        }

        function ItemList(){ //☆컴포넌트 이름 대문자
            //로딩 / 데이터 / 에러 이 3가지를 상태로 관리
            const [users, setUsers] = useState([]); //서버에서 받아올 유저 목록. 초기값.
            const [loading, setLoading] = useState(true); //로딩상태
            const [error, setError] = useState(null); //에러 상태

            useEffect(() => {
                fetch('https://jsonplaceholder.typicode.com/users')
                .then(response => {
                    //fetch문은 if(response.ok) 무조건 체크
                    if(!response.ok){
                        throw new Error('네트워크 응답 문제가 있습니다.');
                    }
                    return response.json();     
                })//fetch가 성공해서 response를 받으면 then이 실행됨.
                //responseUsers: 바로 위 response.json()의 결과값이 자동으로 들어오는 변수
                .then(responseUsers => setUsers(responseUsers)) //= return response.json(); 
                .catch(error => {
                    console.error('데이터를 불러오는 데 실패했습니다.',error);
                    setError(error); //에러 발생 시 에러 상태에 저장
                })
                .finally( () => setLoading(false)); //로딩상태 false로 바꿔준다.
            
            },[]);
            //Early Return 패턴
            if (loading){
                return <p>로딩중...</p>
            }
            if (error){
                return <p>오류가 발생했습니다: {error.message}</p>
            }
            //데이터가 보장된 상태
            return(
                <div>
                    { users.map( user => <Item key={user.id} user={user}/>  )}
                </div>
            )

        }
        function Item({user}){
            return (
                <div className="item">
                    <h4>{user.name}</h4>    
                    userId : {user.id} <br />
                    email : {user.email} <br />
                    phone : {user.phone} <br />
                    company name: {user.company?.name} 
                    {/* 옵셔널 체이닝 연산자 [?] 사용(에러 방지)
                        user.company가 존재하면 → .name 접근해서 출력
                        user.company가 없으면(undefined/null이면) → 그냥 undefined 반환하고 에러 안 냄
                    */}
                </div>
            )
        }

        function App(){
            return(
                <>
                    <Title/>
                    <ItemList />                
                </>
            )
        }
        ReactDOM.createRoot(document.getElementById('root')).render(<App />);

    </script>
</body>
</html>
```
- **입력값 검증**: 검색어 입력 시 `trim()`을 사용해 공백을 제거하고, 데이터 존재 여부를 먼저 확인한 후 대괄호 표기법(`emojis[name]`)으로 접근합니다.
```javascript
// 실습 코드 핵심 패턴
const handleSearch = (emojiName) => {
    // 방어적 코드: 검색어가 없거나 데이터가 없으면 즉시 종료
    if (!emojis || emojiName.trim() === '') {
        setImageUrl(null);
        return;
    }
    // 대괄호 표기법으로 키값 접근
    const foundUrl = emojis[emojiName];
    setImageUrl(foundUrl || null);
};
```
