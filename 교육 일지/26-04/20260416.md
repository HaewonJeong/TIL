### React 특징
- Virtual DOM 이라는 가상 DOM을 사용함
- 기존에는 가장 하위 DOM만 수정해도 전체 DOM이 렌더링 됨. 리액트는 실제 DOM에게 변경사항이 발생한 부분만 업데이트 라고 알려줌
- Component를 사용하여 UI를 구성. 상태(state) 및 속성(props)를 활용하여 컴포넌트 렌더링. (코드의 재사용성 및 개발 생산성 향상)

### React 주요 요소
- 1. 엘리먼트(Element)
- 2. 컴포넌트(Component) : 엘리먼트를 반환하는 **화면의 구성 단위**
  컴포넌트에는 클래스형 컴포넌트, 함수형 컴포넌트가 있음.

      }

3. Virtual DOM
- 모든 변경 사항에 대해 전체 DOM을 업데이트 하는 대신 변경 사항이 발생한 부분만 업데이트하여 DOM 조작
을 최소화 하기 위한 개념
- Render Tree를 참고하여 Virtual DOM이 생성되며 이러한 가상 돔(Virtual DOM) 개념을 통해 애플리케이
션의 성능을 크게 향상 시키고 대규모 애플리케이션 개발에 매우 유용하다.
4. JSX

### React 동작 원리
- HTML Parsing: HTML을 파싱해서 DOM Tree를 생성
- CSS Parsing : CSS 파일을 파싱해서 CSSOM Tree를 생성
- Render : 두개의 Tree를 결합하여 Render Tree 생성
- Layout : Render Tree를 화면에 배치하는 것과 관련하여 각 노드의 크기를 계산하여 화면상에 실제로 픽셀로 변환하고 레이어를 생산
- Paint : 레이어를 생성하여 실제 화면에 표시

### React에서 Virtual Dom을 활용한 Re-Rendering
- 바뀔때 마다 렌더링 X, 모아서 한번만 알려줌
- **리액트는 Rendering Tree를 이용해 가상돔을 생성 한다.**
- **이후 상태(STATE)가 변경 될 때 마다 Re-Rendering이 발생하며 변화가 적용된 새로운 Render Tree로 새로운 가상돔을 생성하게 되고 기존의 가상 돔과의 차이를 비교(Diffing)하여 Browser에 그려질 때 사용될 새로운 Render Tree(적용 될 실제 돔)를 생성하는데 이러한 과정을 재조정(Reconcilication)이라고 한다.**
