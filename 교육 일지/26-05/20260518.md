/* Application 2 : 프로퍼티 파일을 불러와서 JDBC 연결 하는 방법*/
public class Application2 {
    public static void main(String[] args) {

        Properties prop = new Properties(); //설정 파일 전용 Map Collection
        Connection con = null;

        //프로퍼티의 파일의 내용을 읽어서 객체에 받는다.
        try(FileReader reader = new FileReader("src/main/java/com/ohgiraffers/config/connection-info.properties"))
        {
            prop.load(reader); //Properties 파일 내부의 내용을 읽어 prop 객체에 담는다.
            System.out.println("prop!");

            String driver = prop.getProperty("driver"); //key 이름으로 value 값을 꺼내 온다.
            String url = prop.getProperty("url");
            String user = prop.getProperty("user");
            String password = prop.getProperty("password");

            Class.forName(driver);

            con = DriverManager.getConnection(url, user, password);

            System.out.println(con);


        } catch (FileNotFoundException e) {
            e.printStackTrace();
        } catch (IOException e) {
            e.printStackTrace();
        } catch (ClassNotFoundException e) {
            e.printStackTrace();
        } catch (SQLException e) {
            e.printStackTrace();
        } finally {
            if(con != null){
                try {
                    if(!con.isClosed()){
                        con.close();
                    }
                } catch (SQLException e) {
                    e.printStackTrace();
                }
            }
        }
    }

}

## Chapter 02
- preparedStatment와 Stament의 차이
```
코드)
Connection con = getConnection();
PreparedStatement pstmt = null;
ResultSet rset = null;

Statement와 PreparedStatement는 JDBC에서 SQL 쿼리를 실행할 때 사용하는 두 개의 핵심 인터페이스입니다.

작성하신 코드 흐름을 기준으로 두 방식이 내부적으로 어떻게 다르게 작동하는지, 그리고 왜 실무에서는 대부분 PreparedStatement를 쓰는지 명확하게 비교해 드릴게요.

1. 가장 큰 차이점: "컴파일(해석)을 언제 하는가?"
데이터베이스(MySQL)가 자바가 보낸 SQL을 실행하려면 ① 문법 검사 ➡️ ② 구문 분석(컴파일) ➡️ ③ 실행의 단계를 거쳐야 합니다. 이 과정을 처리하는 방식이 완전히 다릅니다.

① Statement (방금 전 에러 났던 방식)
특징: 매번 실행할 때마다 쿼리를 처음부터 끝까지 새로 만듭니다.

동작: 값이 바뀔 때마다 쿼리 문자열 전체가 통째로 바뀌기 때문에, 데이터베이스는 호출할 때마다 매번 1단계부터 3단계(컴파일)를 새로 수행해야 합니다.

비유: 번역가에게 매번 새로운 문장이 적힌 종이를 주며 처음부터 다시 번역하라고 시키는 것과 같습니다.

② PreparedStatement (물음표 ?를 쓰는 방식)
특징: 쿼리의 뼈대(틀)를 미리 컴파일해 둡니다.

동작: con.prepareStatement(query)를 선언하는 순간, 데이터베이스는 물음표(?)가 뚫려있는 뼈대 쿼리를 미리 분석(컴파일)해 둡니다. 그 후 pstmt.setString(1, menuCode)으로 값만 갈아끼워 실행하기 때문에, 실행할 때는 컴파일 과정을 건너뛰고 즉시 실행합니다.

비유: 구멍이 뚫린 서식지 양식을 미리 인쇄해 두고, 손님이 올 때마다 빈칸에 이름만 적어서 바로 처리하는 것과 같습니다.

2. 한눈에 보는 핵심 차이점 비교 테이블
비교 항목	Statement	PreparedStatement
쿼리 형태	완전한 SQL 문자열 ("WHERE CODE = '" + code + "'")	위치홀더 ? 사용 ("WHERE CODE = ?")
컴파일 시점	executeQuery()가 호출될 때마다 매번 컴파일	prepareStatement() 선언 시 최초 1회만 컴파일
실행 속도	반복 실행 시 매번 컴파일하므로 느림	뼈대를 재사용하고 값만 바꾸므로 매우 빠름
따옴표 처리	문자열/날짜형 데이터에 ' '를 직접 감싸야 함	setXXX() 메서드가 타입을 인지하고 자동 처리
보안성	SQL Injection 해킹 공격에 취약함	SQL Injection 공격을 완벽히 차단함
3. 왜 PreparedStatement가 더 안전한가요? (보안)
앞서 설명해 드린 "미리 컴파일을 해둔다"는 특징이 보안에서 엄청난 차이를 만듭니다.

만약 악의적인 사용자가 입력창에 메뉴 번호 대신 6 OR 1=1 같은 조건을 해킹 목적으로 입력했다고 가정해 보겠습니다.

Statement의 경우: 문자열이 그대로 더해져서 "SELECT ... WHERE MENU_CODE = 6 OR 1=1" 이라는 완전히 새로운 해킹 쿼리가 완성되고, 데이터베이스는 이를 그대로 컴파일하여 테이블의 모든 데이터를 유출시키게 됩니다.

PreparedStatement의 경우: 이미 데이터베이스는 "WHERE MENU_CODE = ?" 라는 구조로 컴파일을 끝내놓았습니다. 따라서 사용자가 아무리 이상한 SQL 명령어를 입력해도, 컴퓨터는 이를 코드로 인식하지 않고 "6 OR 1=1"이라는 단순한 '글자(값)' 자체로만 매칭해 버립니다. 결국 해당 글자와 똑같이 생긴 메뉴 코드를 찾다가 안전하게 "조회 결과 없음"을 반환하게 됩니다.

💡 요약
Statement는 매번 쿼리 문장을 새로 조립하고 새로 컴파일하는 단순한 방식입니다.

PreparedStatement는 쿼리의 틀(?)을 미리 짜두어 데이터베이스에 등록해 놓은 뒤, 값만 쏙쏙 바꿔 끼우는 방식입니다. 속도, 가독성, 보안(해킹 방지) 모든 면에서 우수하기 때문에 실무 프로젝트에서는 99% PreparedStatement를 표준으로 사용합니다!
```

public class Application1 {

    //private static String empId = "200";
    //private static String empName = "선동일";

    private static String empId = "210";
    private static String empName = "' OR 1=1 AND EMP_ID = '200"; 
    //SQL Injection 발생
    //query = SELECT * FROM EMPLOYEE WHERE EMP_ID = '210' AND EMP_NAME = '' OR 1=1 AND EMP_ID = '200'
    //id 및 이름이 둘 다 맞아야 출력되는것이 의도이나,
    // OR 1=1 구문이 항상 참이어서, ID 200 사원이 출력됨

    public static void main(String[] args) {

        Connection con = getConnection();
        Statement stmt = null;
        ResultSet rset = null;
        String query = "SELECT * FROM EMPLOYEE WHERE EMP_ID = '" + empId + "' AND EMP_NAME = '" + empName + "'";
        System.out.println("query = " + query);

        try {
            stmt = con.createStatement();
            rset = stmt.executeQuery(query);

            if(rset.next()){
                System.out.println(rset.getString("EMP_NAME")+"님 환영합니다.");
            }else{
                System.out.println("조회 할 정보가 없습니다.");
            }

        } catch (SQLException e) {
            e.printStackTrace();
        } finally {
            close(rset);
            close(stmt);
            close(con);
        }

    }

}
