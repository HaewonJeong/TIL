##### CookietestServlet.java
```
    protected void doGet(HttpServletRequest req, HttpServletResponse resp) throws ServletException, IOException {

        Cookie cookie = new Cookie("rememberId", "user01");
        cookie.setMaxAge(60 * 60 * 24); 
        cookie.setPath("/"); 
        cookie.setHttpOnly(true);
        resp.addCookie(cookie);.
        resp.sendRedirect("redirect");

    }
```

#### SessionestServlet.java
```
   protected void doGet(HttpServletRequest req, HttpServletResponse resp) throws ServletException, IOException {
        //현재 요청에 연결된 세션을 가져온다. (없으면 새로 생성)
        HttpSession session = req.getSession();

        //세션 만료 시간 설정
        session.setMaxInactiveInterval(60 * 10);

        //서버 세션 공간에 loginUser 값 저장
        session.setAttribute("loginUㄴer", "user01");

        resp.sendRedirect("redirect");

    }
```
RedirectResponseServlet.java
```
        //세션 코드
        HttpSession session = req.getSession();
        String loginUser = session == null ? null : (String) session.getAttribute("loginUser");

        if(loginUser == null){
            loginUser = "세션 정보 없음";
        }
```
- 세션의 실제 값은 서버에 저장되고, 브라우저는 JSESSIONID 쿠키만 가지고 다닙니다.
- 브라우저에서는 JesssionId를 확인 할 수 없고 브라우저에서는 서버쪽에 존재한다.

- Cookie
  - 값이 브라우저에 저장
  - 다음 요청에 자동 전송
  - 사용자가 확인하거나 조작 가능
  - 아이디 기억하기, 간단한 설정값 등에 사용 
- Session
  - 실제 값이 서버에 저장
  - 브라우저는 JSESSIONID 쿠키만 저장
  - 로그인 상태 관리에 자주 사용
  - 서버 메모리나 세션 저장소 관리가 필요
