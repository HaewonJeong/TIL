## PROJECTION
	- **SELECT 절에 조회할 대상을 지정**하는 것을 프로젝션이라고 함.
	- **id 값 조회 외에 SPQL SELECT 절에서 원하는 값을 직접 조회** 할 때 사용함.
	- (SELECT {프로젝션 대상} FROM)
	- 1. 엔터티 프로젝션 : 원하는 객체를 바로 조회할 수 있다. 조회된 엔티티는 영속성 컨텍스트가 관리한다. 
		- 예시) String jpql = "SELECT m FROM Section03Menu m";
	2. 임베디드 타입 프로젝션 : 조회의 시작점이 될 수 없다. (from 절 사용 불가) 임베디드 타입은 영속성 컨텍스트에 서 관리되지 않는다. 
		- 
	3. 스칼라 타입 프로젝션 : 숫자, 문자, 날짜 같은 기본 데이터 타입이다. 스칼라 타입은 영속성 컨텍스트에서 관리되 지 않는다. 
		- 스칼라 = 단일 값
		- 숫자, 문자열 하나만 조회. 단일값은 데이터가 아니므로 영속성 컨텍스트에서 관리 되지 않는다.
	4. new 명령어(DTO)를 활용한 프로젝션 : 다양한 종류의 단순 값들을 DTO로 바로 조회하는 방식으로 new 패키지 명.DTO명을 쓰면 해당 DTO로 바로 반환받을 수 있다. new 명령어를 사용한 클래스의 객체는 엔티티가 아니므로 영속성 컨텍스트에서 관리되지 않는다.
  
## Paging  
  
### 개념  
JPA는 DBMS마다 다른 페이징 SQL 문법을 추상화하여 API 형태로 제공한다.  
  
- `setFirstResult(offset)` : 조회 시작 위치 지정  
- `setMaxResults(limit)` : 조회할 최대 데이터 수 지정  
  
### 실습  
  
```java  
String jpql = "SELECT m FROM Section04Menu m ORDER BY m.menuCode DESC";  
  
List<Menu> menuList = entityManager.createQuery(jpql, Menu.class)  
.setFirstResult(offset)  
.setMaxResults(limit)  
.getResultList();
```

### 결과
- offset = 10
- limit = 5

11번째 데이터부터 5개의 데이터를 조회하였다.

## Group Function

JPQL에서도 SQL과 동일하게 그룹 함수를 사용할 수 있다.

### 사용 가능한 함수

- COUNT
- SUM
- AVG
- MAX
- MIN

### 주의사항

- COUNT 결과는 `Long` 타입으로 반환된다.
- SUM, AVG 등의 결과도 `Long` 또는 `Double` 타입으로 반환된다.
- COUNT를 제외한 그룹 함수는 조회 결과가 없으면 `null`을 반환한다.
- HAVING 절의 비교값도 `Long`, `Double` 타입을 사용하는 것이 안전하다.
- JPQL의 그룹함수는 COUNT, MAX, MIN, SUM, AVG로 SQL의 그룹함수와 차이가 없다. 단 몇 가지 주의사항이 있다. 
- 그룹함수의 반환 타입은 결과 값이 정수면 Long, 실수면 Double로 반환된다. 
- 값이 없는 상태에서 count를 제외한 그룹 함수는 null이되고 count만 0이 된다. 따라서 반환 값을 담기 위해 선 언하는 변수 타입을 기본자료형으로 하게 되면, 조회 결과를 언박싱 할때 NPE가 발생한다. 
- 그룹 함수의 반환 자료형은 Long 또는 Double 형이기 때문에 Having 절에서 그룹 함수 결과값과 비교하기 위 한 파라미터 타입은 Long or Double로 해야한다.

## 3. COUNT 사용

### 실습

특정 카테고리에 등록된 메뉴 개수 조회

```
SELECT COUNT(m.menuPrice)FROM Section05Menu mWHERE m.categoryCode = :categoryCode
```

### 결과

카테고리 4의 메뉴 개수를 조회하였다.

---

## 4. SUM 사용

### 실습

```
SELECT SUM(m.menuPrice)FROM Section05Menu mWHERE m.categoryCode = :categoryCode
```

### 확인 내용

조회 결과가 없는 경우 COUNT는 0을 반환하지만 SUM은 `null`을 반환한다.

따라서 반환 타입을 `Long`으로 선언해야 한다.

## 5. GROUP BY, HAVING

### 실습

카테고리별 메뉴 가격 합계를 구하고 합계가 50,000원 이상인 카테고리 조회

```
SELECT m.categoryCode, SUM(m.menuPrice)FROM Section05Menu mGROUP BY m.categoryCodeHAVING SUM(m.menuPrice) >= :minPrice
```

### 결과

- 카테고리 4 : 279000
- 카테고리 6 : 59000

### 학습 내용

- GROUP BY를 사용하여 카테고리별로 데이터를 그룹화하였다.
- HAVING 절을 사용하여 그룹 함수 결과를 조건으로 필터링하였다.
- 여러 컬럼을 조회할 경우 `Object[]` 형태로 반환된다.

## 느낀점

JPA의 Paging API를 통해 DBMS별 페이징 문법 차이를 신경 쓰지 않고 데이터를 조회할 수 있었다. 또한 COUNT, SUM, GROUP BY, HAVING 등의 그룹 함수를 JPQL에서도 SQL과 유사하게 사용할 수 있음을 확인하였다.
