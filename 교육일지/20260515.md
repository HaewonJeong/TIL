1. 고급 쿼리 기법 (Subqueries & Set Operators)
- 단순 조회를 넘어 데이터를 논리적으로 결합하고 상세하게 필터링하는 방법입니다.
- 서브쿼리 (Subqueries): 쿼리 안에 포함된 또 다른 쿼리입니다.
- WHERE 절 서브쿼리: 특정 조건(예: 특정 메뉴의 카테고리)을 실시간으로 계산하여 메인 쿼리에 전달합니다.
- FROM 절 서브쿼리 (파생 테이블): 쿼리 결과를 마치 하나의 테이블처럼 사용하며, 이때 반드시 별칭(Alias)을 지정해야 합니다.
- 상관 서브쿼리: 바깥쪽 쿼리의 값을 안쪽 쿼리가 참조하여 행마다 결과를 계산하는 심화 기법입니다.
- 집합 연산자 (Set Operators): 두 개 이상의 SELECT 결과를 하나로 합칩니다.
- UNION: 중복을 제거하고 합치기, UNION ALL: 중복을 포함하여 모두 합치기.
- 교집합/차집합: MySQL은 별도 연산자가 없으므로 INNER JOIN, LEFT JOIN, IN 연산자를 활용하여 논리적으로 구현합니다.
2. 데이터 조작 및 트랜잭션 (DML & Transaction)
- 실제 데이터를 넣고 수정하며, 그 작업의 안전성을 보장하는 단계입니다.
- DML (Data Manipulation Language) 
- INSERT: 새로운 행 추가. 
- REPLACE를 사용하면 중복 키 발생 시 기존 데이터를 삭제 후 새 데이터로 덮어씁니다.
- UPDATE / DELETE: 기존 데이터를 수정하거나 삭제합니다.
- 반드시 WHERE 절을 통해 대상을 특정해야 합니다.
- 트랜잭션 (Transaction): "All or Nothing". 여러 작업을 하나의 논리적 단위로 묶습니다.
- START TRANSACTION으로 시작하며, COMMIT으로 최종 저장하거나 ROLLBACK으로 작업을 취소하고 이전 상태로 되돌립니다.
3. 구조 정의 및 제약조건 (DDL & Constraints)
- 데이터베이스의 뼈대를 만들고 데이터의 품질(무결성)을 유지하는 규칙들입니다.
- DDL (Data Definition Language): CREATE(생성), ALTER(수정 - ADD, DROP, CHANGE, MODIFY), DROP(삭제), TRUNCATE(초기화) 명령어를 포함합니다.
- 제약조건 (Constraints): 잘못된 데이터가 들어오지 못하게 막는 방어 기제입니다.
- 제약조건 설명 
- PRIMARY KEY : 테이블의 대표 식별자 (NOT NULL + UNIQUE)
- FOREIGN KEY : 다른 테이블을 참조하여 관계를 맺음. 부모 데이터 변화 시 CASCADE나 SET NULL 옵션 설정 가능
- NOT NULL : 빈 값(NULL) 허용 안 함
- UNIQUE : 중복 값 허용 안 함
- CHECK : 설정한 특정 조건(범위 등)만 허용
- DEFAULT : 값이 입력되지 않았을 때 자동으로 채워질 기본값
4. 데이터 타입 및 성능 최적화 (Data Type & Index)
- 형변환: CAST()나 CONVERT() 함수를 통해 데이터 타입을 일시적으로 변경합니다 (예: 평균 가격을 정수로 변환).
- 인덱스 (Index): 데이터 검색 속도를 비약적으로 높여주는 '목차'와 같습니다.
- EXPLAIN을 통해 쿼리가 인덱스를 타는지 실행 계획을 확인할 수 있습니다.
- 단, 과도한 인덱스는 삽입/수정 성능을 떨어뜨리므로 주의가 필요합니다.
