#### 회원 관리 프로그램 객체 지향적인 구조(Application, Service, Repository, DTO)로 나누어 구현
- Member : Entity(or DTO) | 데이터의 주인공: 이름, 나이 같은 정보 그 자체를 담는 바구니
- Service : Service | 비즈니스 로직: "회원을 가입시킨다", "중복을 확인한다" 같은 실제 기능을 수행
- Repository : Repository | 데이터베이스 관리: 데이터를 저장(store)하거나 찾는(find) 창고 관리자
- Application : Controller | 입구 및 창구: 사용자의 명령(메뉴 선택 등)을 받아 적절한 서비스로 연결

### 상속
- 부모의 기능을 모두 물려받는다.
- 상속 방법: public class extend 부모 클래스명 {...}
- super()
  - 부모의 기본 생성자를 호출
  - 생성자의 반드시 가장 첫 줄에 와야 한다.

- 오버라이드 : 메소드 재정의
- 부모의 메소드를 오버라이드를 통해 입맛에 맞게 재정의 가능
- @Override 어노테이션
  - 부모 클래스로부터 물려 받은 메소드를 나에게 맞게 재정의 하는 것을 의미.
  - 이 어노테이션을 붙이면, 컴파일러가 오버라이딩 규칙을 잘 지켰는지 검사해준다.(실수 방지)
  - 꼭 붙이는 습관을 들이면 좋다. 오타 발생 시 빨간줄 그어줌.
  - 오버라이딩 성공 조건
    - 메소드 이름, 반환 타입, 매개변수 목록이 모두 부모와 동일해야 한다.
    - 접근 제한자는 부모와 같거나 더 넓어야 한다.
  
### 다형성
- 하나의 객체나 메서드가 여러 가지 형태를 가질 수 있는 성질
- 상속 관계일때만 가능
- 업 캐스팅 : 자식 객체를 부모 타입의 변수에 할당
```
 Animal a1 = new Cat();
 Animal a2 = new Tiger();
 //a1.jump();    // Animal 참조 변수이기 때문에 Cat만 가진 jump 호출 불가
```
- 다운 캐스팅 : 명시적으로 타입 강제 변환
```
((Cat)a1).jump();
//((Cat)a2).jump(); // 런타임 에러! Tiger -> Cat으로 형변환 불가
```
- 동적 바인딩 : 컴파일 시점에는 부모(Animal)의 메소드를 호출하는 것으로 보이지만, 런타임 시점에는 실제 가리키는 객체의 오버라이딩 된 메소드가 실행되는 원리
``` 
  a1.cry();
  a2.cry();
```
- instanceof: 객체가 특정 클래스(또는 생성자)의 인스턴스인지, 혹은 상속 관계에 있는지를 판별하여 True 혹은 False로 반환해주는 연산자
- 다형성 배열 : 조상타입의 배열에 자손들의 객체를 담을수 있음. 여러종류의 객체를 배열로 다룰 수 있게 됨.

### 추상 클래스(abstract class)
- abstract class, extends
- 메서드의 선언부만 작성하고 구현부는 미완성인 채로 남겨둔 메소드. (메소드의 구현부인 중괄호가 없는 형태)
- 추상 메서드는 본문을 가질 수 없다.
- 추상 클래스를 상속 받는 모든 자식은 반드시 추상 메서드를 직접 구현해야한다.
- 추상 클래스는 인텔리제이 아이콘이 네군데가 뚫려있음
- new 생성자를 통해 인스턴스 객체로 직접 만들 수 없다.
  - 직접적인 인스턴스화가 불가능 하다 뿐이지, super() 메소드를 이용해 추상 클래스 생성자 호출이 가능하다.
- 단일 상속만 가능.
- 변수 선언 시 일반 필드 가능
- 공통 기능 + 강제
### 인터페이스
- interface, implements
- 여러개 구현 가능 - default 메서드로만.
- 변수 선언 시 상수만 가능(static final 키워드 붙음)
- default 메소드 : java 8이 등장하면서 새로 등장. 
  - 인터페이스에 새로운 기능이 추가 했을때 기능을 다 수정해야하는 고통을 피하기 위해 추가됨
  - 인터페이스에 기본 구현 제공한다.
  - 기존 구현체를 수정하지 않고 새 기능을 추가 할 때 사용한다.(하위 호환성)
- 순수 규격/계약 정의

- 추상화, 인터페이스 개념을 알아두면 스프링의 Dependency Injection(의존성 주입) 배울때 도움 많이 된다.
