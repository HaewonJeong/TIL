### JPQL(Java Persistence Query Language)
- JPQL은 엔터티 객체를 중심으로 개발할 수 있는 객체 지향 쿼리
## JPQL이란?

- JPA에서 사용하는 객체 지향 쿼리 언어    
- SQL과 비슷하지만 테이블이 아닌 **엔티티 객체를 대상으로 조회**    
- 특정 DBMS에 의존하지 않음    
- Hibernate가 DB 방언(Dialect)에 맞는 SQL로 변환하여 실행    
- `find()`와 달리 JPQL은 항상 DB에 SQL을 실행하여 조회

---
## 1-1. JPQL 기본 문법

### 특징
- SQL과 동일하게 `SELECT`, `UPDATE`, `DELETE` 사용    
- `INSERT`는 JPQL이 아닌 `persist()` 사용    
- 키워드는 대소문자 구분 없음    
- 엔티티명과 필드명은 대소문자 구분    
- 엔티티 별칭(alias) 사용 필수   

### 작성 순서
1. JPQL 문자열 작성    
2. `entityManager.createQuery()`로 쿼리 생성    
3. `getSingleResult()` 또는 `getResultList()` 실행    

---

## Query 종류

### TypedQuery

반환 타입을 명확하게 지정할 때 사용

```java
TypedQuery<String> query =
    entityManager.createQuery(jpql, String.class);
```

특징

- 타입 안정성 보장    
- 형변환 불필요    

---

### Query

반환 타입을 명확하게 지정할 수 없을 때 사용
```java
Query query = entityManager.createQuery(jpql);
```

특징
- 결과가 Object 또는 Object[] 형태로 반환    

---

## 결과 조회 메소드

### getSingleResult()
- 결과가 정확히 1건일 때 사용    
- 결과가 없거나 2건 이상이면 예외 발생    

```java
String menuName = query.getSingleResult();
```

---

### getResultList()
- 여러 건 조회 시 사용    
- 결과가 없으면 빈 컬렉션 반환    

```java
List<Menu> menuList = query.getResultList();
```

## JPQL과 SQL 차이

### SQL

```sql
SELECT *
FROM tbl_menu
```

- 테이블과 컬럼 기준
    

### JPQL

```java
SELECT m
FROM Section01Menu m
```

- 엔티티와 필드 기준
    

### 주의

JPQL에서는

```sql
SELECT *
```

사용 불가

반드시

```java
SELECT m
```

또는

```java
SELECT m.menuName
```

형태로 작성해야 한다.
## 1-2. parameter JPQL 
- 구문에 파라미터를 바인딩 하는 경우 두 가지 방법이 있다. 
- 첫 번째는 **이름 기준 파라미터(named parameters) 바인딩**으로 **:** 다음에 이름 기준 파라미터를 지정한다. 

- 두 번째는 **위치 기준 파라미터(positional parameters) 바인딩**으로 **?** 다음에 위치 값을 주는데 위치 값은 1부터 시작한다. 

- 테스트를 위한 Menu 엔터티는 1-1. 기본 문법에서 사용했던 구조를 따르되 Entity name만 변경한다.
