### Next JS 특징
- 1. 서버 사이드 렌더링(SSR) 지원
Next는 server html을 만들어서 보내게 지원. 검색 엔진에 노출이 잘 됨.
React는 기본적으로 브라우저에서 JS 실행 후 화면을 그리는데,
Next.js는 서버에서 HTML을 먼저 만들어서 브라우저에 보내줄 수 있다.
그래서 초기 로딩이 빠르고, SEO(검색 노출) 에 유리함.

- 2. 파일 기반 라우팅
페이지가 자동으로 연결 됨
pages 또는 app 폴더 안에 파일을 만들면 그 파일 경로가 그대로 URL이 된다.

- 3. 정적 사이트 생성(SSG) 과 하이브리드 렌더링
페이지 성격에 따라 미리 만들어놓거나 or 필요할때만
Next.js는 페이지를 상황에 따라 다르게 렌더링할 수 있다.

- SSG: 빌드할 때 HTML을 미리 만들어둠 (빠름)
- SSR: 요청 올 때마다 서버에서 HTML 생성
- CSR: 브라우저에서 React가 렌더링
즉, 페이지마다 “미리 만들지 / 요청마다 만들지” 선택 가능

- 4. 풀스택 개발 환경
FE 뿐만 아니라 apI server도 만들 수 있다.
Next.js는 프론트만 만드는 게 아니라
/api 같은 경로로 백엔드 API도 같이 만들 수 있어요.

- 결론. 즉 Next는 프론트 화면, 서버 기능(API), 배포까지 한 프로젝트 안에서 같이 관리 가능한 구조
- 프론트엔드 실무 지향이면 Next 배우는 게 매우 유리함

```

  return (
    <>
      <h1>메인 페이지 입니다!</h1>
      <nav>

    )

```

### Next.js에서 제공하는 "라우팅 관련 Hook" 정리
즉, URL 읽기 + URL로 이동하기를 담당하는 Hook.
- usePathname() 현재 URL 경로를 읽어옴 -> 활성 탭/메뉴 스타일 적용 시
- useParams() [폴더명] 형태의 동적 라우팅 파라미터를 읽어온다 -> 상세페이지(/menu/1)
- useSearchParams() URL 끝의 쿼리 스트링(?key=value)를 읽어옴 -> 검색 결과 페이지, 필터링
- useRouter() 함수/이벤트 내에서 프로그래밍 방식으로 페이지 이동 -> 폼 제출 완료 페이지 이동, 뒤로가기
- 'use client' 클라이언트 컴포넌트에서만 사용 가능
