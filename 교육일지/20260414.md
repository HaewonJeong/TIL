## 1. DOM 요소 선택 (Selector)
HTML 문서를 자바스크립트가 제어할 수 있는 '객체'로 가져오는 과정

### 1-1. 현대적인 선택 방법 (권장)
- **`querySelector()`**: CSS 선택자를 사용하여 **첫 번째** 요소 하나만 반환합니다.
- **`querySelectorAll()`**: 일치하는 **모든** 요소를 `NodeList`(정적 컬렉션) 형태로 반환합니다.

### 1-2. 과거 방식과 주의점
- `getElementById`, `getElementsByClassName` 등은 `HTMLCollection`을 반환하는데, 이는 **실시간(Live) 컬렉션**입니다. 
- 반복문 도중에 요소의 클래스를 바꾸면 목록이 실시간으로 변해 의도치 않은 버그(건너뛰기 등)가 발생할 수 있습니다.
- **해결책**: 스프레드 문법(`[...]`)을 사용하여 진짜 **배열**로 변환 후 사용하는 것이 안전 합니다.

---

## 2. 노드 탐색 (Traversal)
찾은 요소를 기준으로 부모, 자식, 형제 노드로 이동하는 방법입니다.

- **자식 찾기**: `children`(요소만), `firstElementChild`, `lastElementChild`
- **부모 찾기**: `parentNode`
- **형제 찾기**: `nextElementSibling`, `previousElementSibling`
> **Tip**: `childNodes`나 `nextSibling`은 눈에 보이지 않는 **공백이나 줄바꿈(텍스트 노드)**까지 포함하므로, 가급적 **`Element`**가 붙은 프로퍼티를 사용하는 것이 좋습니다.

---

## 3. DOM 조작 (Manipulation)

### 3-1. 텍스트 조작
- **`.textContent`**: 요소 안의 텍스트를 읽거나 쓸 때 사용합니다. HTML 태그는 무시하고 오직 '글자'만 다룹니다.
- **`.innerHTML`**: HTML 태그를 직접 해석해서 넣습니다. 간편하지만 사용자의 입력을 그대로 넣을 경우 **보안(XSS) 공격**에 취약하므로 주의해야 합니다.

### 3-2. 노드 생성 및 추가 (레고 조립)
1. `createElement()`: 새로운 태그 만들기
2. `textContent` 등으로 내용 채우기
3. `appendChild()` 또는 `insertBefore()`: 실제 화면(DOM)에 붙이기

> **성능 최적화**: 여러 요소를 반복문으로 추가할 때는 `DocumentFragment`라는 **임시 조립판**에 먼저 다 붙인 뒤, 마지막에 딱 한 번만 실제 화면에 붙이는 것이 성능상 훨씬 유리합니다.

## 4. 어트리뷰트(Attribute) vs 프로퍼티(Property)
이 부분이 처음엔 헷갈릴 수 있는데, **'설계도'**와 **'실제 상태'**의 차이입니다.

- **Attribute**: HTML 문서에 적힌 **초기 상태** (설계도). `getAttribute()`, `setAttribute()`로 접근.
- **Property**: 자바스크립트 객체가 가진 **현재 상태** (실시간). `.value`, `.checked` 등으로 직접 접근.
- **사용자 입력값**: 사용자가 입력한 최신 값을 가져올 때는 반드시 **프로퍼티(`.value`)**를 사용해야 합니다.

## 5. CSS 조작하기

- **인라인 스타일**: `$el.style.backgroundColor = 'blue';` (우선순위가 너무 높아 관리가 힘듦)
- **클래스 제어 (권장)**: 미리 CSS 파일에 클래스를 정의해두고 자바스크립트로 클래스만 뗐다 붙였다 합니다.
  - `$el.classList.add('active');`
  - `$el.classList.remove('active');`
  - `$el.classList.toggle('active');` (있으면 지우고, 없으면 추가)
