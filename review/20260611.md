##### @Embeddable, Embedded 어노테이션
- Embedded를 붙인 테이블이 따로 생기지 않고 속한 테이블 안에 컬럼이 생성 된다.  
- @Embeddable : 포함 될 클래스. 값 객체 클래스에 붙임. 테이블 생성 안 함.
- @Embedded : 엔티티 안에 포함되는 값 객체
- 사용 방법
```
@Embeddable
class Address {}

@Entity  
public class Product {    
	@Embedded  
	private Price price;  
  }
```

자바 변수명(camelCase)을 DB 컬럼명(snake_case)으로 자동 변환해주는 규칙  
physical-strategy: org.hibernate.boot.model.naming.CamelCaseToUnderscoresNamingStrategy

#### Association Mapping
- N:1(ManyToOne) 연관 관계
- 1:N(OneToMany) 연관 관계
- 1:1(OneToOne) 연관 관계
- N:N(ManyToMany) 연관 관계 <- DB에서는 N:N 직접 표현 불가능. 객체 세계에서는 가능.

#### cascade = CascadeType.PERSIST

``@ManyToOne(cascade = CascadeType.PERSIST)`는

```
@ManyToOne(cascade = CascadeType.PERSIST)@JoinColumn(name = "category_code")private Category category;
```

처럼 사용해.

의미는

```
Menu를 저장할 때(PERSIST)연관된 Category도 같이 저장해라
```

야.

## 그림으로

### Cascade 없음

```
Menu └ Category(신규 객체)persist(menu)     ↓에러
```

---

### Cascade PERSIST 있음

```
Menu └ Category(신규 객체)persist(menu)     ↓persist(category)     ↓persist(menu)     ↓성공
```

# 2. @Entity

JPA가 관리하는 엔티티 클래스임을 나타낸다.

```java
@Entity
public class Member {
}
```

또는

```java
@Entity(name = "MemberEntity")
```

- 데이터베이스 테이블과 매핑
- 기본 생성자 필수
- final 클래스 사용 불가
- 저장 필드에 final 사용 불가

---

# 3. @Id, @GeneratedValue

기본 키(PK)를 지정하고 자동 생성 전략을 설정한다.

```java
@Id
@GeneratedValue(strategy = GenerationType.IDENTITY)
private Long id;
```

## 생성 전략

| 전략 | 설명 |
|--------|--------|
| IDENTITY | MySQL AUTO_INCREMENT |
| SEQUENCE | Oracle Sequence |
| TABLE | 별도 키 생성 테이블 사용 |
| AUTO | DB 환경에 따라 자동 선택 |

---

# 4. @Column

엔티티 필드와 DB 컬럼을 매핑한다.

```java
@Column(name = "member_name")
private String memberName;
```

주요 속성

| 속성 | 설명 |
|--------|--------|
| name | 컬럼명 지정 |
| nullable | NULL 허용 여부 |
| unique | 유니크 제약조건 |
| length | 문자열 길이 |
| insertable | INSERT 포함 여부 |
| updatable | UPDATE 포함 여부 |

---

# 5. @Transient

DB 컬럼과 매핑하지 않는 필드

```java
@Transient
private int age;
```

- 데이터베이스에 저장되지 않음
- 조회 시에도 값이 존재하지 않음
- 계산용 필드에 사용

---

# 6. Enum 매핑

## Enum 정의

```java
public enum RoleType {
    ADMIN,
    MEMBER
}
```

## @Enumerated 사용

```java
@Enumerated(EnumType.STRING)
private RoleType role;
```

### EnumType.ORDINAL

```java
ADMIN -> 0
MEMBER -> 1
```

장점
- 저장 공간 적음

단점
- Enum 순서 변경 시 문제 발생

### EnumType.STRING

```java
ADMIN -> "ADMIN"
MEMBER -> "MEMBER"
```

장점
- Enum 추가/순서 변경에 안전

단점
- 저장 공간 증가

실무에서는 대부분 STRING 사용

```java
@Enumerated(EnumType.STRING)
```

## Chap04. Association Mapping 학습 정리

## 1. 연관관계 매핑(Association Mapping)

JPA에서 연관관계 매핑은 엔티티(Entity) 간의 관계를 객체로 표현하는 방법이다.

### 다중성에 따른 분류

- N:1 (ManyToOne)
    
- 1:N (OneToMany)
    
- 1:1 (OneToOne)
    
- N:N (ManyToMany)
    

### 방향에 따른 분류

- 단방향 연관관계
    
- 양방향 연관관계
    

---

# 2. ManyToOne (N:1)

## 개념

여러 개의 Menu가 하나의 Category를 참조하는 관계이다.

```java
@ManyToOne
@JoinColumn(name = "categoryCode")
private Category category;
```

### 주요 어노테이션

#### @ManyToOne

- 다대일 관계 설정
    
- Menu → Category 참조
    

#### @JoinColumn

- 외래키(FK) 컬럼 지정
    
- Menu 테이블의 category_code 와 Category를 연결
    

---

## 객체 그래프 탐색

```java
Menu menu = repository.find(menuCode);

Category category = menu.getCategory();
```

객체를 통해 연관된 엔티티를 조회할 수 있다.

---

## JPQL 조회

```java
SELECT c.categoryName
FROM menu_and_category m
JOIN m.category c
WHERE m.menuCode = :menuCode
```

특징

- FROM 절에는 테이블명이 아닌 엔티티명 사용
    
- 객체 중심 쿼리 작성
    

---

## Cascade Persist

```java
@ManyToOne(cascade = CascadeType.PERSIST)
```

의미

- Menu 저장 시 Category도 함께 저장
    
- 부모(Category) → 자식(Menu) 순으로 INSERT 수행
    

---

# 3. OneToMany (1:N)

## 개념

하나의 Category가 여러 개의 Menu를 가지는 관계이다.

```java
@OneToMany
@JoinColumn(name = "categoryCode")
private List<Menu> menuList;
```

---

## 객체 그래프 탐색

```java
Category category = repository.find(categoryCode);

List<Menu> menus = category.getMenuList();
```

카테고리를 통해 소속 메뉴 목록을 조회할 수 있다.

---

## Cascade Persist

```java
@OneToMany(cascade = CascadeType.PERSIST)
```

의미

- Category 저장 시 Menu들도 함께 저장
    

실행 순서

1. Category INSERT
    
2. Menu INSERT
    
3. FK UPDATE
    

---

# 4. Fetch 전략

## LAZY (지연 로딩)

기본 전략

```java
@OneToMany(fetch = FetchType.LAZY)
```

동작

1. Category 조회
    
2. menuList 사용 시 Menu 조회
    

장점

- 필요한 시점에만 조회
    
- 성능 최적화
    

---

## EAGER (즉시 로딩)

```java
@OneToMany(fetch = FetchType.EAGER)
```

동작

- 조회 시 JOIN을 이용하여 함께 조회
    

장점

- 추가 조회 없음
    

단점

- 불필요한 데이터까지 조회 가능
    

---

# 5. 양방향 연관관계(Bidirectional)

## 개념

객체는 양방향 참조를 위해 양쪽에 필드를 가져야 한다.

### 연관관계 주인

외래키(FK)를 가진 엔티티가 연관관계의 주인이다.

예)

- Menu → Category 참조
    
- Menu가 FK 보유
    
- Menu가 연관관계의 주인
    

---

## 주인 엔티티

```java
@ManyToOne
@JoinColumn(name = "categoryCode")
private Category category;
```

---

## 주인이 아닌 엔티티

```java
@OneToMany(mappedBy = "category")
private List<Menu> menuList;
```

### mappedBy

- 연관관계의 주인을 지정
    
- 실제 FK를 관리하지 않음
    
- 조회 전용 역할
