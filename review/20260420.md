### 1) Promise
- 비동기 작업(서버 요청, 타이머 등)의 결과를 “나중에” 받을 수 있게 해주는 객체입니다.
- 성공(resolve)하거나 실패(reject)하는 상태를 가집니다.
- .then()으로 성공 결과를 받고 .catch()로 에러를 처리합니다.

### 2) async / await
- Promise를 더 읽기 쉽게 작성하도록 만든 문법입니다.
- async 함수는 항상 Promise를 반환합니다.
- await는 Promise가 끝날 때까지 기다렸다가 결과를 변수에 저장합니다.
- 따라서 .then()을 여러 번 쓰지 않아도 코드를 깔끔하게 작성할 수 있습니다.
- 에러 처리는 보통 try/catch 문으로 처리합니다.

### 3) useRef
- useRef는 React에서 값을 저장하는데, 값이 바뀌어도 렌더링을 다시 하지 않는 저장소입니다.
- 즉, state처럼 화면에 영향을 주는 값이 아니라 “기억만 해야 하는 값”에 사용합니다.
- 또한 DOM 요소에 직접 접근할 때 자주 사용됩니다(예: input focus).
- useRef는 { current: 값 } 형태의 객체를 반환합니다.
- ref.current에 값이 저장되며 렌더링이 반복되어도 ref 값은 초기화되지 않고 유지됩니다.

### 4) useMemo
- 어떤 계산 결과를 저장(캐싱)해서 불필요한 재계산을 막는 Hook입니다.
- 의존성 배열 값이 바뀔 때만 다시 계산합니다.
- 무거운 연산이 반복되어 화면이 느려질 때 주로 사용합니다.

### 5) useCallback
- useCallback은 함수를 저장해서 렌더링마다 새 함수가 만들어지는 것을 방지합니다.
- React는 컴포넌트가 다시 렌더링되면 내부 함수도 새로 생성됩니다.
- 부모가 자식에게 함수를 props로 넘기면, 함수가 바뀐 것으로 판단되어 자식이 재렌더링될 수 있습니다.
- useCallback은 의존성 배열이 바뀌지 않으면 같은 함수 객체를 유지합니다.
- 즉, “함수 재생성 방지”를 통한 렌더링 최적화 목적입니다.
