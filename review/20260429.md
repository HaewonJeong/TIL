### 패키지와 클래스 차이점
```
📁 1. 패키지 (Package) : "코드 정리용 폴더"
패키지는 관련된 자바 파일들을 예쁘게 모아두는 폴더입니다.
역할: 컴퓨터 바탕화면에 파일이 1,000개 흩어져 있으면 찾기 힘들죠? 그래서 '사진', '문서', '게임' 폴더를 만들어 정리하는 것처럼, 자바에서도 성격이 비슷한 코드들을 모아두기 위해 패키지를 만듭니다.
특징: 이름이 겹치는 것을 막아줍니다. 예를 들어 A팀이 만든 Application1과 B팀이 만든 Application1이 있어도, 패키지(폴더)가 다르면 에러가 나지 않고 평화롭게 공존할 수 있습니다.
인텔리제이 생김새: 일반적인 폴더 아이콘 모양입니다. (예: com, ohgiraffers, section03, constant)

📝 2. 클래스 (Class) : "코드가 적힌 파일"
클래스는 우리가 실제로 컴퓨터에게 내릴 명령(코드)을 적어두는 파일이자 도화지입니다.
역할: 프로그램이 실제로 어떻게 동작할지, 어떤 계산을 할지 구체적인 내용(소스 코드)을 작성하는 곳입니다.
특징: 자바 프로그램은 무조건 하나 이상의 클래스로 이루어져야 합니다. 관례상 클래스 이름은 항상 대문자로 시작합니다.
인텔리제이 생김새: 파일 이름 옆에 초록색 동그라미 모양의 C 아이콘이 붙어있습니다. (예: Application1, Application2)
```

### 비교 연산자
- Java는 '===' 연산자 없음
- System.out.println("str 논리값 비교 "+str1 == str2); 출력 시 'str 논리값'이 앞에 출력 안됐다. 더하기(+) 연산자가 비교(==) 연산자보다 먼저 실행되기 때문
- 자바에서 == 연산자는 글자 내용이 아니라 '메모리 주소(위치)'가 같은지를 비교 한다. 문자열 값 비교를 하고 싶을때는 .equals()를 사용

- Java에서 stack, heap
```aiignore
public void exampleMethod() {
    int age = 30;                     // (1) 원시 타입
    String name = new String("Java"); // (2) 참조 타입 (객체)
}

(1) int age = 30;

age라는 변수 이름과 30이라는 실제 값이 모두 Stack에 예쁘게 쌓입니다.

(2) String name = new String("Java");

new 키워드를 썼죠? "Java"라는 실제 데이터(객체)는 Heap(물류 창고)에 큼지막하게 자리를 잡습니다.

Stack에는 name이라는 변수가 생기는데, 여기에는 실제 글자가 아니라 "Heap 창고의 몇 번 구역에 데이터가 있는지" (메모리 주소)만 저장됩니다. 즉, 스택에서 힙을 향해 화살표를 쏘고 있는 상태입니다.

(3) 메서드 종료 (})

메서드가 끝나는 순간, 책상 위 포스트잇(Stack)은 한 번에 싹 다 지워집니다. (age와 name 변수 소멸)

하지만 Heap에 있던 "Java" 객체는 창고에 덩그러니 남습니다.

이제 "Java" 객체를 가리키는 화살표(포스트잇)가 세상에 하나도 없으므로, 잠시 후 Garbage Collector가 "어? 이거 이제 아무도 안 쓰네?" 하고 창고에서 치워버립니다.
```
