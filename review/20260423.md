1. 물과 기름의 만남: 'use client'와 async

에러: <Movie> is an async Client Component. Only Server Components can be async...

이유: 리액트(Next.js)에서 'use client'를 쓰기로 마음먹었다면, 함수 이름 앞에 async를 절대 붙일 수 없습니다. async는 서버 컴포넌트만의 특별한 권한입니다.

해결: export default async function Movie()에서 async 단어를 지워주세요.

왜 Movie 컴포넌트에는 async를 못 붙이게 했을까요?
이유는 클라이언트 컴포넌트('use client')의 엄청나게 급한 성격 때문입니다.

리액트는 사용자가 페이지에 들어오자마자 0.001초 만에 일단 껍데기(빈 화면)라도 무조건 먼저 그려서 보여줘야 직성이 풀립니다.
그런데 Movie 컴포넌트 자체에 async를 걸어버리면, 리액트는 **"영화 API 데이터가 다 올 때까지 화면 전체를 멈추고 하얗게 대기"**해야 합니다. 리액트 입장에서는 견딜 수 없는 일이죠. 그래서 클라이언트 컴포넌트에는 async를 달 수 없도록 아예 문법으로 막아버린 것입니다.

🛠️ 컴포넌트에 async를 안 붙이고 진동벨(Promise)을 처리하는 2가지 방법
회원님이 겪으신 이 딜레마를 해결하기 위해 리액트 개발자들은 두 가지 방법을 씁니다.

1. 구식 진동벨 .then() 사용하기 (우리가 아까 쓴 방법)
부모 함수에 async/await을 쓸 수 없으니, 예전에 쓰던 방식인 .then을 꺼내온 것입니다.

JavaScript
useEffect(() => {
    // "목록 가져와! 다 가져오고 나면(then), 결과(res)를 바구니에 담아(setMoiveList)"
    getMovieList().then(res => setMoiveList(res));
}, []);
이 방식은 컴포넌트를 멈추지 않고, 일단 빈 화면을 먼저 그린 다음 뒤에서 조용히 데이터를 받아와 화면을 업데이트합니다.

2. useEffect 안에 '심부름꾼 함수' 만들기 (현업에서 가장 많이 쓰는 꿀팁 🍯)
"나는 .then 쓰기 싫어! 무조건 최신 문법인 await을 쓸 거야!" 라고 하신다면, 이렇게 하시면 됩니다.
useEffect 안쪽에 async를 달 수 있는 작은 심부름꾼 함수를 하나 몰래 만들어서 걔한테 시키는 겁니다.

JavaScript
useEffect(() => {
    // 1. async를 붙인 심부름꾼 함수를 하나 만듭니다. (얘는 컴포넌트가 아니라서 async 합법!)
    const fetchMovies = async () => {
        const res = await getMovieList(); // 여기서 await 사용 가능!
        setMoiveList(res);
    };

    // 2. 심부름꾼 출발!
    fetchMovies();
}, []);
결론:
getMovieList()가 Promise를 반환하는 것은 정확히 맞습니다! 다만, 화면을 당장 그려야 하는 리액트의 규칙 때문에 최상단 컴포넌트에는 async를 붙일 수 없어서, .then()을 쓰거나 useEffect 안에서 따로 심부름꾼을 만들어 처리하는 것입니다.
