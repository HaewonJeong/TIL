## React 비동기 통신 및 API 연동 실습 일지

### 1. 비동기 통신의 기초 (Fetch API)
`fetch`는 브라우저 내장 API로, 서버에 데이터를 요청하고 응답을 받는 가장 기본적인 방법입니다.

### 1-1. Promise와 JSON 파싱
- `fetch` 호출 시 즉시 데이터가 오는 것이 아니라 **Promise(미래의 응답)** 객체가 반환됩니다.
- 응답 본문을 읽으려면 `await response.json()`을 통해 자바스크립트 객체로 변환하는 과정이 필요합니다.

```
async function callAPI() {
    // 1. 서버에 요청 (Promise 반환)
    const response = await fetch('[https://jsonplaceholder.typicode.com/users](https://jsonplaceholder.typicode.com/users)');
    
    // 2. 응답 본문을 JSON으로 파싱
    const data = await response.json();
    console.log(data);
}
```

### 2. 비동기 처리 문법 비교
비동기 코드를 제어하는 두 가지 주요 방식을 실습했습니다.

### 2-1. .then() 체이닝
콜백 함수를 연결하여 순차적으로 실행하는 방식입니다.

성공(then)과 실패(catch)를 명확히 구분하여 처리합니다.

### 2-2. async / await (권장)
비동기 코드를 마치 동기 코드(위에서 아래로)처럼 읽히게 작성할 수 있어 가독성이 뛰어납니다.

try-catch 블록을 사용하여 에러를 직관적으로 관리합니다.

### 3. Axios 라이브러리
fetch의 불편함을 개선한 외부 라이브러리인 Axios를 활용했습니다.

### 3-1. Axios의 주요 장점
자동 JSON 파싱: .json() 과정 없이 response.data로 바로 데이터에 접근합니다.

HTTP 에러 처리: 404, 500번대 에러 발생 시 자동으로 catch 블록으로 이동합니다.

직관적인 메서드: axios.get(), axios.post() 등으로 가독성이 높습니다.

### 4. React 컴포넌트 데이터 연동
실제 React 컴포넌트 내에서 데이터를 가져와 화면에 그리는 실무 패턴을 익혔습니다.

### 4-1. 3가지 핵심 상태 관리
API 통신 시 사용자 경험(UX)을 위해 다음 세 가지 상태를 반드시 관리해야 합니다.

데이터(data): 서버로부터 받은 실제 결과값

로딩(loading): 통신 중임을 알리는 상태 (스피너 표시 등)

에러(error): 통신 실패 시 사용자에게 보여줄 에러 정보

### 4-2. useEffect를 활용한 데이터 호출
컴포넌트가 처음 나타날 때(Mount) 딱 한 번만 데이터를 가져오도록 의존성 배열([])을 설정합니다.

방어적 코드: 데이터가 도착하기 전 에러를 방지하기 위해 **옵셔널 체이닝(?.)**을 사용합니다.

```
function ItemList() {
    const [users, setUsers] = useState([]);
    const [loading, setLoading] = useState(true);

    useEffect(() => {
        axios.get('[https://jsonplaceholder.typicode.com/users](https://jsonplaceholder.typicode.com/users)')
            .then(res => setUsers(res.data))
            .finally(() => setLoading(false));
    }, []);

    if (loading) return <p>데이터를 불러오는 중입니다...</p>;

    return (
        <div>
            {users.map(user => (
                <div key={user.id}>
                    <h4>{user.name}</h4>
                    <p>{user.company?.name}</p> {/* 옵셔널 체이닝 사용 */}
                </div>
            ))}
        </div>
    );
}
```
