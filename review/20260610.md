#### JPA 이어서 실습
- EntityManagerFactory : EntityManager를 생성하는 공장 객체  
- EntityManager : 엔티티를 조회, 저장, 수정, 삭제하는 객체 
- Persistence Context : 엔티티를 저장하고 관리하는 메모리 공간으로 변경 감지 기능을 제공한다.

JPA 구조
- Persist contex DB는 분리된 공간이다. Persist context는 메모리 상 있는 공간으로, DB와는 분리된 공간. commit까지 해야 DB 수정이 일어난다.

#### Chapter 01 - Section03
- 엔티티 생명주기
- EntityManager가 존재하는 동안 생명주기를 지칭하는 용어
	- 비영속(new/transient) : 엔티티가 영속성 컨텍스트와 전혀 관계가 없는 상태 
	- 영속(managed) : 엔티티가 영속성 컨텍스트에 저장된 상태 
	- 준영속(detached) : 영속성 컨텍스트에 저장되었다가 분리된 상태 
	- 삭제(removed) : 엔티티가 삭제된 상태 
	- 병합(merge) : 엔티티가 준영속 상태인 엔티티가 다시 영속상태로 변경된 상태
#### 준영속 엔티티의 PK를 변경한 뒤 merge()하면, JPA는 해당 PK의 엔티티를 찾을 수 없다고 판단하여 UPDATE가 아니라 INSERT를 수행할 수 있다.
- detached 된 상태로 20 -> 999 하고, merge하면?  
- DB, Persist Context에도 값이 없어 새로 insert하는 상황이 일어난다.  
- 준영속 엔티티의 PK를 변경한 뒤 merge()하면, JPA는 해당 PK의 엔티티를 찾을 수 없다고 판단하여 UPDATE가 아니라 INSERT를 수행할 수 있다.
```
// DB에 존재하는 메뉴menuCode = 20menuName = "김치찌개"
```

JPA 자바 ORM 표준 인터페이스
- ORM 객체 관계 매핑 -> 자바 객체 & DB 테이블 
- 보편적으로 많이 사용되는 구현체 :  Hibernate
- 장점
	- 1. 캐싱 (1차 캐시)
		- JPA는 조회한 엔티티를 **영속성 컨텍스트(Persistence Context)** 에 저장한다.
		- 같은 `EntityManager`에서 동일한 PK를 다시 조회하면 DB에 `SELECT`를 보내지 않고 영속성 컨텍스트에 저장된 객체를 반환한다.
		- 따라서 DB 접근 횟수가 줄어들어 성능이 향상된다.
	- 2. 유지보수성 / 생산성 증가
		- CRUD SQL문을 직접 작성할 필요가 없음
		- `persist()`, `find()`, `remove()` 등으로 데이터 조작 가능하다.

```
entityManager.persist(menu);
entityManager.find(Menu.class, 1);
entityManager.remove(menu);
```
    - 3. 객체 중심 개발
		- SQL보다 객체(Entity)를 중심으로 개발한다.
		- 테이블의 컬럼을 직접 다루기보다 자바 객체를 조작하면 된다.
```
menu.setMenuPrice(10000);
```
↓
JPA가 알아서
```
UPDATE tbl_menuSET menu_price = 10000WHERE menu_code = ?;
```
를 생성한다.
	  - 4.  `persist()`, `find()`, `remove()` 등으로 데이터 조작 가능하다.

```
entityManager.persist(menu);entityManager.find(Menu.class, 1);entityManager.remove(menu);
```

##### 1. 1차 캐시

- 영속성 컨텍스트 내부에서 엔티티를 캐싱한다.
- Key는 PK(@Id), Value는 엔티티 객체이다.
- 같은 PK를 다시 조회하면 DB 조회 없이 캐시에서 가져온다.

##### 2. 동일성 보장

- 같은 EntityManager에서 동일한 PK를 조회하면 같은 객체를 반환한다.

```
menu1 == menu2 // true
```

---

##### 3. 트랜잭션을 지원하는 쓰기 지연

- INSERT, UPDATE, DELETE SQL을 즉시 DB에 보내지 않는다.
- SQL을 쓰기 지연 저장소에 모아두었다가 commit 시 한 번에 실행한다.

---

##### 4. 플러시(Flush)

- 영속성 컨텍스트의 변경 내용을 DB에 반영한다.
- flush(), commit(), JPQL 실행 시 자동 수행된다.
- 단, 트랜잭션은 종료되지 않아 rollback 가능하다.

---

#### 5. 변경 감지(Dirty Checking)

- 영속 상태 엔티티의 변경을 자동으로 감지한다.
- 엔티티와 스냅샷을 비교하여 UPDATE SQL을 생성한다.
- 영속 상태 엔티티에만 적용된다.

```
menu.setMenuPrice(10000);
```

↓

```
UPDATE tbl_menu ...
```

##### Entity 설정과 실제 생성된 DB 결과 비교
```
package com.ohgiraffers.mapping.section01.entity;  
  
import jakarta.persistence.*;  
  
import java.time.LocalDateTime;  
  
//Entity 이름을 지정. 만약 작성하지 않으면 class 이름을 엔티티 명으로 사용한다.  
@Entity(name = "entityMember")  
@Table(name = "tbl_member")  
public class Member {  
  
    //PK 지정. 지정하지 않으면 have primary key~ 에러 남  
    @Id  
    @Column(name = "member_no")  
    //자동 생성 전략을 지정. 스프링은 GenerationType.IDENTITY로 전략 가져가기.  
    @GeneratedValue(strategy = GenerationType.IDENTITY)  
    private int memberNo;  
    @Column(  
            //컬럼 제약 조건 unique : 고유값, nullable = false : notnull 등..  
            name = "member_id", unique = true,  
            nullable = false, columnDefinition = "varchar(10)"  
    )  
    private String memberId;  
    @Column(name = "member_pwd", nullable = false)  
    private String memberPwd;  
    @Column(name = "member_name")  
    private String memberName;  
  
    //phone 컬럼을 매핑에서 제외  
    @Transient  
    @Column(name = "phone")  
    private String phone;  
  
    //length 컬럼의 길이  
    @Column(name = "address", length = 900)  
    private String address;  
    @Column(name = "enroll_date")  
    private LocalDateTime enrollDate;  
  
    @Column(name = "member_role")  
    //Enum을 String/숫자으로 다룰지 선택. 선택하지 않으면 숫자 사용한다.  
    //ORDINAL(숫자) 방식은 Enum 순서 변경 시 데이터 오류가 발생할 수 있어    // 실무에서는 주로 STRING 방식을 사용한다.    @Enumerated(EnumType.STRING)  
    private MemberRole memberRole;  
    @Column(name = "status", columnDefinition = "char(1) default 'Y'")  
    private String status;  
  
    protected Member() {  
    }  
  
    public Member(  
            String memberId, String memberPwd, String memberName,  
            String phone, String address, LocalDateTime enrollDate,  
            MemberRole memberRole, String status  
    ) {  
        this.memberId = memberId;  
        this.memberPwd = memberPwd;  
        this.memberName = memberName;  
        this.phone = phone;  
        this.address = address;  
        this.enrollDate = enrollDate;  
        this.memberRole = memberRole;  
        this.status = status;  
    }  
}
