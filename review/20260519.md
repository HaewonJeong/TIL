### JDBC
#### JDBC Update, Detele
- executeUpdate()의 result 의미
executeUpdate()는 보통 이렇게 리턴해:
1 : 업데이트 성공 (1행 변경됨)
0 : 조건에 맞는 행이 없어서 아무것도 변경 안 됨
-1 : JDBC에 따라 거의 안 나옴 (보통 안 씀)
즉, 실패(변경 안 됨) 상황은 result == 0

#### 카디널리티(Cardinality)
- N:1 관계 예시
```
짜장면 2    1 한식
짬뽕   2    1 한식
    N    :    1
```
- 1:이 부모, N이 자식 테이블이 됨
- 부모의 (PK) 가 자식으로 넘어가서 외래키(FK)가 됨
#### 관계(Relationship)
- 참조 관계 : Join을 위해서 만드는 관계. 연관성 없이 만드는 관계
- 배타 관계
    - 직원이 고객이랑 관계를 맺으면 관계가 끊어진다.
    - 자식(N)이 부모(1)가 둘일때, 하나만 연결 될 수 있다.

### 논리 모델
#### 정규화
- 정규화란
  - 중복 데이터를 제거하여 안정성과 확장성을 얻음
함수 종속
- 결정자 : Primary Key.(X) ex.메뉴 코드
- 종속자 : X가 결정되면 따라오는 Y
- 대표 속성이 나머지 속성을 유일하게 식별할 수 있다면 대표 속성과 나머지 속성 사이에는 연관 관계가 성립하고 이것을 함수 종속이라고 한다.

- 정규화 
  - 실무에서는 거의 3정규형까지 함
- 1 정규형(1NF)
  - 반복 속성을 제거. 원자값을 가진다.
  - 모든 속성은 하나의 값을 가져야한다. (원자값을 지녀야 한다.)
  - 인조 식별자
- 2 정규형 (2NF)
  - 후보 식별자 속성과 일반 속성 간의 종속성을 판단하여 처리한다.
- 3 정규형 (3NF)
  - 이행적 종속성을 제거하고 일반 속성간에 종속 관계가 없도록 처리한다.
