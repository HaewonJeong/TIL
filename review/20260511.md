#### 1. Comparable/  오름차순 정렬
```
        System.out.println("Comparable/  오름차순 정렬");
        Collections.sort(bookList); //sort 구현해야 작동 됨
        for(BookDTO book : bookList){
            System.out.println(book);
        }
```
1. 객체 정렬의 전제 조건
Collections.sort(List<T> list) 메서드는 매개변수로 받는 리스트의 요소 타입(T)이 반드시 Comparable 인터페이스를 구현하고 있어야만 작동합니다. 내부적으로 Timsort 알고리즘을 사용하며, 이 알고리즘은 정렬 과정에서 요소 간의 선후 관계를 결정하는 비교 연산이 필요하기 때문입니다.

2. 내부 동작 흐름 (Stack Trace 관점)
Collections.sort(bookList)를 호출하면 다음과 같은 순서로 실행됩니다.
타입 확인: 자바 컴파일러와 런타임은 BookDTO가 Comparable<BookDTO>를 implements 했는지 확인합니다.
비교 메서드 호출: 정렬 알고리즘이 리스트 내의 두 객체(예: book1, book2)를 선택합니다.
메서드 실행: 알고리즘 내부에서 book1.compareTo(book2)를 호출합니다.

반환값 분석:
반환값이 음수이면: book1이 book2보다 작다고 판단하여 앞에 배치합니다.
반환값이 양수이면: book1이 book2보다 크다고 판단하여 뒤에 배치합니다.
반환값이 0이면: 두 객체의 우선순위가 같다고 판단하여 현재 순서를 유지합니다.

#### 3. Comparator 익명 클래스 사용 / 내름차순 정렬
```
        //익명 클래스 사용(가격 내림차순)
        bookList.sort(new Comparator<BookDTO>() {
            @Override
            public int compare(BookDTO o1, BookDTO o2) {
                return Integer.compare(o2.getPrice(), o1.getPrice());
            }
        });
```

### List
- list : 순서가 있고, 중복을 허용한다.
#### LinkedList
- LinkedList 시간복잡도
  - 조회 : O(n) 다음 노드->다음 노드를 찾아기 때문. index 조회가 많을때는 ArrayList가 낫다.
  - 맨 앞 삽입/삭제 : O(1)
  - 중간 삽입/삭제 : O(n)
  - 맨 뒤 삽입/삭제 : O(1)
       
- ArrayList 시간 복잡도
  - 조회 : O(1)
  - 맨 앞 삽입/삭제 : O(n)
  - 중간 삽입/삭제 : O(n)
  - 맨 뒤 삽입/삭제 : O(1)

#### Queue
- Queue: 선입선출(FIFO) - 먼저 넣은 것이 먼저 나온다.
- Queue는 인터페이스이므로 new 키워드로 생성 불가하다.
- LinkedList로 구현체 생성
- 1. queue.offer : 추가
- 2. queue.peek : 첫번째 요소 조회(제거x)
- 3. queue.poll : 첫번째 요소 조회 및 제거
- 4. 빈 queue에서 poll 하는 경우 null 반환

#### Stack과 Queue의 차이
- 123 -> 3 (Stack은 123 넣었다 꺼내면 3 나온다.). 후입 선출(LIFO)
- 123 -> 1 (queue는 123 넣었다 꺼내면 1 나온다.) 선입 선출(FIFO)

### Set
- set : 순서 보장하지 않음. 중복 허용 하지 않음.
- 중복이면 무시되고 false 반환
```
        Set<String> hset = new HashSet<>();
        hset.add("html");
        hset.add("java");
        hset.add("css");
        boolean isAdded = hset.add("java"); //중복이면 무시되고 false 반환

        System.out.println(hset);
        System.out.println(isAdded);

        hset.add("java"); //중복 추가는 무시
```

#### 구현체 1 HashSet : 순서 없음 + 중복 불가
- 중복된 요소를 삽입하면 반영되지 않는다. 그리고 처음 삽입한 요소의 순서가 유지 된다.
#### 구현체 2 LinkedHashSet : 중복 불가 + 순서 유지
- 중복된 요소를 삽입하면 반영 되지 않지만, HashSet과 다르게 처음 삽입한 중복 요소의 순서가 뒤로 간다.

### Map
#### HashMap
- Map 인터페이스
  - key와 value를 하나의 쌍(Entry)으로 묶어서 관리
  - Key는 중복 저장을 허용하지 않는다.
  - value는 중복 저장을 허용한다.
  - Json 데이터 구조 Key-value 와 비슷한 자료구조
- get(key) : 특정 키로 값 조회
- get(key) : 존재하지 않는 key로 get() 하면 null값 반환
- containsKey(key) :실무에서의 방어 코드
- getOrDefault("없는 키", "기본값") : 키 값이 없다면 null이 아니고 전달한 기본값을 반환
- remove(key) : 키-값 쌍 삭제


#### Properties
- HashMap을 상속받은 특수 map
- key와 value를 모두 반드시 String
- 주 용도 : 프로그램 설정 정보(DB 연결 정보, 환경설정 값) 관리
- setProperty(String key, String value)
  - key-value 쌍 형태로 삽입한다. 단 String 형태만 가능하다.
- getProperty(String key)
  - key값으로 꺼낸다.
