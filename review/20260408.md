## JavaScript 데이터 타입
### 1. 변수(Variables)
- 변수는 데이터를 저장할 때 쓰이는 '이름이 붙은 저장소' 이다.
- 변수를 생성할 때 우리는 변수를 선언한다고 표현한다.
- 키워드 변수명;
- 키워드 : 변수의 접근 범위를 지정
- 변수명 : 변수를 부를 이름 설정.
- **var** : ES5까지 사용했던 유일한 키워드. 변수 재선언과 재할당이 모두 가능. 예전에는 많이 사용했지만 지금은 잘 사용 안하고, 아직도 이전 코드에서 가끔 보임.
```
var number;
var number = 5;
```
- **let** : 요즘 사용하는 변수. 값을 변경(재할당)할 수 있는 변수 선언할 때 사용
```
let greeting = "Hello, Node.js!";
```

- **const** : 재할당이 금지된 상수를 선언할때 사용, 반드시 선언과 동시에 초기화 필요

  - 변수 이름은 알파벳, 숫자, _, $만 사용 가능 (숫자로 시작 불가)

  - !의미 있는 이름 사용: 변수 이름은 그 변수의 역할이나 내용을 명확하게 나타내는것이 좋다. 최대한 변수의 역할을 생각해서 작성하는 습관을 가지자.
 ```

### 원시 데이터 타입(Primitive types)
- JavaScript의 데이터 타입은 크게 변경 불가능한 원시 타입(Primitive type)*과 객체 타입으로 나뉜다.
  - 숫자(Number), 문자열(string), 불린(boolean), null, undefined 등이 있다.

```

let age = 30;
console.log(age);

// 숫자 타입은 모두 실수로 처리되므로 둘은 같다.
console.log(10 === 10.0);
console.log(10 / 4);

// 2. 문자열
// 작은 따옴표(''), 큰 따옴표(""), 또는 백틱(``)으로 텍스트를 감싼다.

let string2 = '문자열';
let string3 = `문자열`;
console.log(string1);

// 3. 불린(boolean) : 논리적 참, 거짓을 나타내는 true와 false 뿐이다.

console.log(isStudent);

// 4. null: 명시적으로 값이 없음을 의도적으로 명시할 때 사용.

console.log(address);

// 5. undefined: 변수에 값이 할당되지 않았을 때 자동으로 할당되는 값
let salary;
console.log(salary);

// 6. typeof : 데이터 타입을 확인 할 때 사용

### 3. 타입 변환(Type Coersion)
####  암묵적 타입 변환
- 자바스크립트 엔진에 의해 자동으로 타입이 변환되는 현상
```
문자열 변환: 10 + '20' → '1020' (더하기 연산자 시 한쪽이 문자열이면 문자열 연결)
숫자 변환: 10 * '5' → 50, +'10' → 10
불리언 변환: 조건문 내에서 0, '', null, undefined, NaN은 Falsy한 값으로 평가됩니다.
```

#### 명시적 타입 변환
- 개발자가 의도적으로 타입을 변경하는 것
```
String(10), (10).toString()
Number('10'), parseInt('10.01')
Boolean('JS') → true, !!' ' → true
```

#### JavaScript에서 Falsy한 값으로 평가되는 값
```
 * false
 * 0 (숫자 0)
 * ' ' (빈 문자열)
 * null
 * undefined
 * Nan (Not a Number)
```

### 4. Operator 연산자
#### 산술 연산자
```
//1. 산술 연산자
let a = 10;
let b = 3;

console.log(a + b);
console.log(a - b);
console.log(a * b);
console.log(a / b); //나눗셈
console.log(a % b); //나머지
console.log(a ** b); //거듭제곱

//2. 할당 연산자
let c = 5;
c += 3; //복합 할당 연산자 c = c+3과 같다.
console.log(c); //8

c -= 2; // c = c - 2
console.log(c);

//3. 증감 연산자
let d = 5;
console.log(d++); // 후위 증가
console.log(++d); // 전위 증가
console.log(d--); // 후위 감소
console.log(d);
```

#### 비교 연산자
- **동등 연산자 (==, !=)**: 먼저 암묵적 타입 변환을 통해 타입을 일치시킨 후 같은 값인지 비교
```
console.log(x == y); // ==: 값이 같으면 true (출력: false)
console.log(x != y); // !=: 값이 다르면 true
console.log(x == '10'); //암묵적 타입 변환을 했기 때문에 true 반환

```
- **일치 연산자 (===, !==)**: 타입과 값이 모두 일치하는지 비교
  - 왠만하면 일치 연산자 사용 권장
```
 console.log(x === 10); //값과 타입이 같으면 true
 console.log(x === '10');
```
- **대소 비교 연산자**
```
 console.log(x > y);
 console.log(x >= y);
```
  - 문자열도 크기 비교 가능. 유니코드 순서로 비교. 대문자가 소문자보다 앞에 옴. 이런 비교가 가능하다고 알고 넘어가기.
```
 console.log(`'cat' >= 'Zoo' : ${'cat' >= 'zoo'}`);
 > 'cat' >= 'Zoo' : false
```
#### 논리 연산자와 단축 평가
- **&& (AND)** : 모두 참일 때 참
```
console.log(a && b);
```
- **|| (OR)** : 하나라도 참이면 참
```
console.log(a || b);
```
- **! (NOT)** : 참이면 거짓으로, 거짓이면 참으로 반전
```
console.log(!a);
console.log(!b);
```

- **단축 평가**: 표현식 평가하는 도중 평가 결과가 확정 된 경우 나머지 평가 과정을 생략하는 것
- false && 'banana'는 앞이 거짓이므로 뒤를 확인하지 않고 바로 false를 반환합니다. 이를 이용해 null 체크 에러를 방지할 수 있습니다.
```
console.log('apple' || 'banana'); //apple
console.log(false || 'banana'); //banana
console.log(false && 'banana'); //false
```
- 단축평가를 이용한 Null 체크
- 단축 평가 하여 타입 에러 발생
```
let obj = null;
//let objValue = obj.value; // TypeError: Cannot read properties of null (reading 'value')
```
- obj && obj.value (obj가 null이면 뒤쪽을 실행하지 않음)
```
let objValue = obj && obj.value;
/**obj는 null 값으로, falsy한 값. 그래서 obj 값을 먼저 봤을때 falsy한 값이면, 뒤쪽 obj.value는 확인 안함 > 에러 안남 */
```

#### 삼항 연산자(Ternary Operator)
- **Optional Chaining (옵셔널 체이닝)** : 옵셔널 체이닝(optional chaining)은 객체의 중첩된 속성이나 메서드에 접근할 때, null 또는 undefined 에러를 방지하기 위한 방법
```
const obj1 = null;
//obj가 null 또는 undefined 인 경우 undefined 반환
//obj가 다른 Falsy한 값(0, '')인 경우 그대로 0 또는 '' 반환
//const objValue = obj?.value;
const objValue1 = obj1?.value
```
#### NULL 병합 연산자
- 에크마 스크립트 2020 ES11버전(ECMAScript 2020 ES11) 부터 도입. 좌항의 피연산자가 null 또는 undefined인 경우 우항을 반환, 그렇지 않으면 좌항의 피연산자를 반환 한다.
- >> 변수에 값이 없으면 우항 값을 사용해줘!
```
let test = null ?? '기본 값';
console.log(test);
let value1 = '' ?? '기본 값';
console.log(value1);
let value2 = 0 ?? '기본 값';
console.log(value2);
```
#### 삼항 연산자(Ternary Operator)
- 삼항 연산자는 조건식에 따라 두 값 중 하나를 선택하는 연산자이다.
- 조건 ? 값1 : 값2
- 조건이 true 일 경우 값1, false일 경우 값2 선택
```
let age = 20;

//1. 삼항 연산자 사용
let result = (age >= 20) ? "성인" : "미성년자"; //삼항 연산자
console.log(result);

//2. 중첩된 삼항 연산자
//중첩해서 많이 쓰진 않지만 사용할 수 있긴 하다~
let score = 85;
let grade = (score >= 90) ? "A" : 
    (score >= 80) ? "B" : 
        (score >= 70) ? "C" : "D";
```

### 5. 제어문 (Control Flow)
#### 조건문
- **if...else if...else**: 조건에 따른 분기 처리
- **switch**: 하나의 변수에 대해 여러 case를 비교 (각 case 끝에 break 필수)
#### 반복문
- **for**: 명확한 횟수만큼 반복할 때 사용합니다.
```
for(let i = 1; i <= 5; i++) {
    console.log(`반복 횟수: ${i}`);
}
```
