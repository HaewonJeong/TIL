#### Qualifier
@Qualifier("pikachu")  
- 같은 타입의 빈이 중복될 때, 어떤 빈을 사용할지 지정
```
@Service("pokemonServiceQualifier")
public class PokemonService {

    @Autowired // 👈 "스프링아, 네가 관리하는 포켓몬 중에 하나 여기다 쏙 꽂아줘!"
    @Qualifier("pikachu")  //Pikachu빈을 주입하겠다.
    private Pokemon pokemon;

    public void pokemonAttack(){
        pokemon.attack();
    }

}
```

#### 어노테이션 정리
- @Component : Spring이 관리할 일반 Bean
- @Controller : 웹 요청 처리 계증 bean
- @Service : 비즈니스 로직 계층 bean
- @Repository : 데이터 접근 계층 bean
- @Autowired : 필요한 의존 객체를 자동으로 주입
- @Primary : 기본 우선 Bean 지정
- @Qualifier : 이름으로 특정 Bean 지정
- @Configuration : 스프링 설정 클래스

#### Bean 생명주기
- Bean 생성 > 의존성 주입 > 초기화 > 사용 > 소멸
- initMethod
	- 빈의 라이프사이클(생성 이후 준비 완료 시점, 그리고 소멸 직전 시점)에 맞춰 자동으로 실행될 메서드를 지정하는 설정
	- initMethod : 초기화 메서드(bean이 생성되고 필요한 준비가 끝난 뒤 실행되는 메서드)
	- destroyMethod : 소멸 메서드(Container가 종료되면서 Bean이 정리되기 직전에 실행되는 메서드)
```
    @Bean(initMethod = "openShop", destroyMethod = "closeShop")
    public Owner owner(){
        return new Owner();
    }
```

#### 요약
##### Bean Scope
-singleton : container 안에서 하나만 만들어지고 공유
-prototype : getBean() 호출 할 때마다 새로 만들어짐

##### Bean life Cycle
- Bean은 생성, 의존성 주입, 초기화, 사용, 소멸 흐름을 가진다.
- initMethod, destoryMethod로 특정 시점에 메소드를 실행 할 수 있다.
- @PostConstruct, @PreDestroy 방식도 있다. (참고)

##### 외부 설정값
- 변경 가능성이 있는 값은 Java 코드에 직접 박아두지 않는 것이 좋다.
- properties 파일에 값을 작성하고 @PropertySource, @Value로 읽을 수 있다.


### 스프링 프레임워크 3대 핵심 개념
- IoC, DI, Aop
	- IoC (제어의 역전): new 노가다 끝! 객체의 생성부터 소멸까지 모든 주도권을 스프링이 다 가져가서 대신 관리해 주는 것.
	- DI (의존성 주입): 내가 필요한 부품(객체)을 변수에 @Autowired만 붙여두면, 스프링이 지 창고에서 찾아다가 알아서 쏙 꽂아주는 것.
	- AOP (관점 지향): 메서드마다 복붙하던 귀찮은 반복 코드(로그, 시간 측정 등)를 한곳에 모아두고 원하는 곳에 자동으로 끼워 넣어 실행하는 것.

#### AOP <-이번에 배울 것
핵심 관심사(실제 업무 기능)
- 공통 관심사(로그, 트랜잭션 처리 등 여러곳에서 반복되는 기능)
- AOP: 공통 관심사를 핵심 로직에서 분리하고, 필요한 실행 지점에 자동으로 끼워넣는 방식
- @Aspect  : AOP용 클래스 표시
- @Pointcut (감시 구역 설정) : 내가 전교를 다 감시할 순 없잖아? 내 감시 범위(Pointcut)는 section01 패키지 안에 있고, 이름이 Service로 끝나는 클래스의 모든 메서드들이야!" 하고 감시 구역 렌즈를 딱 고정
- @Before("LoggingAspect.logPointcut()") (감시 타이밍) : 방금 위에서 정한 감시 구역(logPointcut) 안에서, 대상 메서드가 실행되기 바로 직전(Before)에 내가 짠 감시 코드를 먼저 실행할게!" 라는 뜻
